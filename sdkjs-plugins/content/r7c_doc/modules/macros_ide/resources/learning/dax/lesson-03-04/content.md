# Урок 12. FILTER и работа с множествами

Вступление

Представьте библиотекаря, который должен найти книги по сложному запросу: “Все романы XIX века французских авторов, кроме Дюма, изданные в твердом переплете после 1950 года”. Простой поиск по одному критерию здесь не поможет — нужна система сложных фильтров, работающих вместе. В DAX функция FILTER выполняет роль такого универсального библиотекаря, способного отбирать данные по любым сложным условиям, создавая виртуальные таблицы для последующих вычислений.

FILTER и связанные функции работы с множествами открывают новое измерение в DAX — возможность создавать и манипулировать виртуальными таблицами прямо в формулах. VALUES извлекает уникальные значения, DISTINCT убирает дубликаты, HASONEVALUE проверяет единственность выбора, ALLSELECTED работает с видимым контекстом. Эти функции превращают CALCULATE в хирургический инструмент точечной фильтрации.

В предыдущем уроке мы освоили CALCULATE и базовые модификаторы контекста. Теперь расширим арсенал инструментами для создания сложных условий и работы с динамическими множествами.

Теоретическая часть

Основные понятия

```dax
FILTER — это функция-итератор, создающая отфильтрованную копию таблицы на основе логического условия. FILTER проходит по каждой строке исходной таблицы, проверяет условие и включает строку в результат, если условие истинно. Результат FILTER — это виртуальная таблица, которая существует только во время вычисления и может использоваться как аргумент в CALCULATE или итераторах. Например, FILTER(Product, Product[Color] = “Red” && Product[ListPrice] > 100) вернет таблицу только с красными товарами дороже 100 долларов.
VALUES возвращает таблицу с одним столбцом, содержащую уникальные значения из указанного столбца в текущем фильтровом контексте. Если в контексте нет видимых значений, VALUES добавляет строку с BLANK, если есть связанные пустые значения. VALUES(Customer[Country]) вернет список стран только тех клиентов, которые видны в текущем контексте. Это ключевая функция для динамического получения списков значений.
```

DISTINCT похожа на VALUES, но никогда не добавляет строку BLANK и работает с выражениями, а не только столбцами. DISTINCT удаляет дубликаты из таблицы или столбца, возвращая только уникальные строки. DISTINCT(Product[Category]) вернет список категорий без повторений. Разница с VALUES проявляется при работе со связанными таблицами и пустыми значениями.

ALLSELECTED возвращает все значения, видимые в текущем запросе, игнорируя фильтры строк и столбцов сводной таблицы, но сохраняя внешние фильтры и срезы. Это позволяет вычислять проценты от видимого итога, а не от абсолютного итога. ALLSELECTED критична для создания интерактивных отчетов с корректными процентами.

Ключевая механика

Механизм работы FILTER основан на построчной итерации с проверкой условия в строковом контексте. Для каждой строки входной таблицы FILTER создает строковый контекст, вычисляет логическое выражение и решает, включать строку в результат. Это позволяет использовать сложные условия, недоступные в простых булевых фильтрах CALCULATE. FILTER всегда возвращает таблицу, даже если это пустая таблица без строк.

Представьте FILTER как контролера на входе в клуб с списком критериев. Каждый посетитель (строка) проверяется по всем критериям: возраст, дресс-код, наличие приглашения. Только те, кто соответствует всем условиям, попадают внутрь (в результирующую таблицу). Контролер может проверять сложные комбинации: “старше 21 года И в деловой одежде ИЛИ VIP-гость любого возраста”. Простые фильтры CALCULATE как автоматические турникеты — быстрые, но негибкие.

CALCULATETABLE работает аналогично CALCULATE, но возвращает таблицу, а не скалярное значение. CALCULATETABLE модифицирует контекст и возвращает отфильтрованную таблицу. Это полезно для создания промежуточных таблиц в сложных вычислениях. Разница с FILTER: CALCULATETABLE изменяет контекст, FILTER работает в существующем контексте как итератор.

Взаимодействие FILTER с CALCULATE создает мощные комбинации. FILTER как аргумент CALCULATE позволяет применять сложные условия, недоступные через простые булевые выражения. CALCULATE(SUM(Sales[Amount]), FILTER(Product, Product[ListPrice] &gt; Product[StandardCost] * 2)) вычислит продажи только высокомаржинальных товаров. Без FILTER такое условие невозможно выразить.

HASONEVALUE проверяет, содержит ли столбец ровно одно уникальное значение в текущем контексте. Это критично для создания контекстно-зависимых мер. Если выбран один продукт, можно показать детальную информацию, если несколько — агрегированную статистику. HASONEVALUE возвращает TRUE/FALSE и часто используется с IF для ветвления логики.

Синтаксис и правила

Базовый синтаксис FILTER: FILTER(table, boolean_expression). Первый параметр — таблица для фильтрации. Второй — логическое выражение, вычисляемое для каждой строки. В выражении можно ссылаться на столбцы таблицы напрямую, так как создается строковый контекст. Пример: FILTER(Customer, Customer[YearlyIncome] &gt; 50000 &amp;&amp; Customer[TotalChildren] &gt; 2).

```dax
VALUES и DISTINCT имеют простой синтаксис: VALUES(column) или DISTINCT(table_or_column). VALUES работает только со столбцами, DISTINCT может принимать таблицы. Обе функции учитывают текущий фильтровый контекст и возвращают только видимые уникальные значения. Результат — одноколоночная таблица.
CALCULATETABLE имеет синтаксис, аналогичный CALCULATE: CALCULATETABLE(table_expression, filter1, filter2, …). Отличие — первый параметр должен возвращать таблицу, а не скалярное значение. Модификаторы фильтров работают так же, как в CALCULATE. Пример: CALCULATETABLE(Customer, Customer[Country] = “USA”, ALL(Date)).
```

HASONEVALUE принимает один параметр — столбец для проверки: HASONEVALUE(column). Возвращает TRUE, если в текущем контексте столбец содержит ровно одно уникальное значение, иначе FALSE. Часто используется в паттерне: IF(HASONEVALUE(Product[Category]), VALUES(Product[Category]), “Multiple”).

```dax
ALLSELECTED имеет несколько вариантов: ALLSELECTED(table), ALLSELECTED(column), ALLSELECTED(). Без параметров возвращает все данные модели с учетом внешних фильтров. Это сложная функция с нюансами поведения в разных контекстах визуализации.
```

Сравнение с альтернативами

В Excel для сложной фильтрации используются расширенные фильтры или формулы массива. Но они статичны — критерии фиксированы при создании. FILTER в DAX динамически адаптируется к контексту и может использовать вычисляемые условия. Одна формула с FILTER заменяет десятки вариантов SUMIFS.

SQL WHERE и HAVING близки к FILTER по функциональности, но применяются на разных этапах запроса. FILTER в DAX универсален — работает и с исходными данными, и с агрегированными результатами. Это делает DAX более гибким для интерактивной аналитики.

Pandas в Python предлагает похожий функционал через boolean indexing и методы query(). Но в pandas фильтрация явная в коде, в DAX она интегрирована с контекстом вычислений. Это делает DAX-формулы компактнее для аналитических задач.

Частые ошибки

Использование FILTER без необходимости замедляет вычисления. Простые условия равенства лучше выражать через булевые фильтры в CALCULATE. FILTER нужен только для сложных условий, сравнений между столбцами или итерации. Правило: если условие можно выразить без FILTER — не используйте его.

Забывание о строковом контексте в FILTER приводит к ошибкам. Внутри FILTER нельзя использовать меры напрямую — они требуют контекстного перехода через CALCULATE. Выражение FILTER(Sales, [Total Sales] &gt; 1000) некорректно, нужно агрегировать или использовать столбцы.

Путаница между VALUES и DISTINCT создает неожиданные результаты с BLANK. VALUES может добавить строку BLANK, DISTINCT — никогда. При подсчете уникальных значений это может дать разные результаты. Выбирайте функцию осознанно.

Неправильное использование ALLSELECTED в сложных визуализациях дает непредсказуемые результаты. ALLSELECTED чувствительна к контексту визуализации и может вести себя по-разному в таблицах, матрицах и диаграммах. Тщательно тестируйте меры с ALLSELECTED.

Ответы

FILTER создает строковый контекст и позволяет сложные условия между столбцами, булевые фильтры работают только с константными условиями.

VALUES и DISTINCT возвращают одноколоночную таблицу с уникальными значениями из текущего контекста.

```dax
VALUES добавляет BLANK, если есть строки факта без соответствия в измерении (нарушенная ссылочная целостность).
```

HASONEVALUE проверяет, выбрано ли ровно одно значение в столбце, используется для условной логики в мерах.

FILTER — итератор, работающий в текущем контексте; CALCULATETABLE модифицирует контекст и возвращает таблицу.

```dax
ALLSELECTED сохраняет внешние фильтры и срезы, но игнорирует фильтры визуализации (строки/столбцы).
```

FILTER выполняет построчную итерацию, что затратнее простых операций фильтрации на движке.
