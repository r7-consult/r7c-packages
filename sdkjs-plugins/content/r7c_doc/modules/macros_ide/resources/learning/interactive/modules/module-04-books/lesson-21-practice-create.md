# Урок 21. Практика Фабрики листов: Динамическое создание и скрытность

Этот урок — боевое применение теории (Из Урока 19) по управлению листами книги в JavaScript API Р7-Офис. Если вы научитесь проверять существование вкладок и генерировать их "на лету" без человеческого вмешательства, ваша корпоративная автоматизация перейдет на совершенно другой, инженерный уровень надежности (Fault Tolerance).

## 🎯 Бизнес-задача
Вы написали кадровый скрипт, обрабатывающий зарплаты. Раз в год, 1 января, скрипт должен сбрасывать данные за прошлый год в новую вкладку "Архив-[Год]".
Представьте: менеджер заходит в старый файл, нажимает "Заархивировать". Что должен сделать скрипт?
Если он тупо вызовет команду `Api.AddSheet("Архив-2026")`, а этот лист УЖЕ существует (например, кто-то другой нажал кнопку вчера), то макрос "упадет" с ошибкой "Имя листа уже занято", и весь процесс остановится (Exceptions). Это плохой дизайн.

Отличный дизайн: Макрос сначала **проверяет** каждый лист в книге. И только **если не нашел нужного имени**, он его создает, прячет (чтобы не мозолил глаза) и оставляет первую запись: "Секретная База".

## 📝 Постановка задачи (LeetCode Style)

В редакторе стартового кода (справа) уже задана пара переменных: мы запросили все листы книги (`Api.GetSheets()`) и установили переключатель `needNewSheet = true`.

Ваша задача дописать 2 блока кода:
**Блок 1 (Проверка).**
В подготовленном цикле `for` вам необходимо перебрать массив `allSheets` (начинающийся с 0 до `.length`). С помощью метода `.GetName()` на каждом `allSheets[i]` сравните текстовое имя с искомой строкой: `"Архив-2026"`. 
Если сравнение успешно (`===`), переключите флаг `needNewSheet` в позицию `false` и сразу же остановите бессмысленную дальнейшую проверку остальных листов командой `break;`.

**Блок 2 (Фабрика).**
Если цикл завершился, а флаг `needNewSheet` так и остался `true` (значит, лист не найден), запустится второе условие `if(needNewSheet === true)`. 
Внутри него вам надо:
1. Вызвать механизм разработчика `Api.AddSheet("Архив-2026")`.
2. Временно спрятать его для пользователя (Метод `.SetHidden(true)`).
3. Используя метод получения ячейки `.GetRange("A1")` на новом созданном листе, выбейте гравировку: запишите туда значение (строку) `"Секретная База"`. 

Нажмите кнопку "Проверить" ("Run"). Система автотестов Р7 создаст невидимую рабочую книгу, попробует прогнать ваш код и проверит, появился ли в ней спрятанный лист с названием "Архив-2026" и секретной записью!

## 🧠 Глубинный разбор: Флаги состояний (State Flags)

**Почему мы используем переменную, вроде `needNewSheet = true;` и какие-то `break` внутри цикла, вместо того, чтобы просто создавать в `else`?**

Это классическая ошибка многих программистов (и даже аналитиков Python)!
Представьте, что мы пытаемся искать без флага, сразу с `else`:
```javascript
// НЕВЕРНЫЙ И ОЧЕНЬ ВРЕДНЫЙ ПОДХОД
for (let i = 0; i < allSheets.length; i++) {
    if (allSheets[i].GetName() === "Архив-2026") {
        console.log("Лист найден!");
    } else {
        // Мы попали сюда на листе "Лист1" (Сводка)
        Api.AddSheet("Архив-2026"); 
        // И тут макрос падает с ошибкой, потому что Архив-2026 
        // мог оказаться следующим по счету листом, который мы еще не проверили!
    }
}
```

Нельзя делать выводы после прочтения только ОДНОЙ страницы книги. Нужно прочитать все оглавление (весь массив `allSheets`) до конца, и только потом принимать решение о создании.
Именно для этого существует переменная (Флаг состояния). Мы "поднимаем" флаг (запоминаем) перед стартом, прочесываем всю базу. Если нашли — "опускаем" флаг (`false`) и выходим (`break`). Когда цикл завершается, мы смотрим на наш флаг в конце. Если он поднят (всё еще `true`), значит за весь прогон мы так и не встретили искомую вкладку.
Смело создавайте её (Factory).

## ✍️ Эталонное решение (Архитектура)

Если вы запутались в синтаксисе:

```javascript
/* === ЧАСТЬ 1: ЦИКЛ ПРОВЕРКИ === */
for (let i = 0; i < allSheets.length; i++) {
    let currentSheet = allSheets[i];
    let sheetName = currentSheet.GetName();
    
    // Сравниваем
    if (sheetName === "Архив-2026") {
        needNewSheet = false; // Опускаем флаг (Лист уже есть)
        break; // Досрочно убиваем цикл (экономим ресурсы процессора)
    }
}

/* === ЧАСТЬ 2: СОЗДАНИЕ И СКРЫТИЕ === */
if (needNewSheet === true) {
    // В JS можно объединить создание переменной и "возврат" (return) созданного листа 
    let myNewSheet = Api.AddSheet("Архив-2026");
    
    // Скрываем его от интерфейса (пользователь его не увидит в панели вкладок)
    myNewSheet.SetHidden(true);
    
    // Отправляем туда надпись (цепочка вызовов методов)
    myNewSheet.GetRange("A1").SetValue("Секретная База");
}
```

*Обратите внимание: `Api.AddSheet()` — это метод глобального контекста, мы вызываем его у объекта `Api`. А методы `SetHidden` и `GetRange` мы вызываем у локального новорожденного листа (переменной) `myNewSheet`.*
