# Урок 24. Практика Инженерии: Сверхбыстрая фильтрация сделок (Array 2D)

Разработчики VBA, привыкшие работать со `Scripting.Dictionary` или удалять строки прямо на листе (`Rows(i).Delete`), знают, насколько медленно работает удаление в цикле. При каждой итерации пересчитываются адреса нижележащих ячеек. Это приводит к зависанию редактора.

Истинная серверная оптимизация (что в Excel, что в Р7-Офис) — это полная обработка массивов в ОЗУ (ETL-логика), с последующей единовременной выгрузкой через `SetValue`.

## 🎯 Бизнес-задача: VIP-чеки

В базе за месяц 20 000 чеков (строки x колонки). Маркетологу нужны только "VIP" чеки (свыше 10 000 рублей). 

В скрипте справа вы видите получение всего диапазона сделок в переменную `matrix` одной командой `GetValue()`. 
Переменная `matrix` является 2D-массивом вида:
`[ ["Машина", 500000, "Игорь"], ["Хлеб", 50, "Анна"] ]`

**Вам необходимо написать фильтр:**
1. Перебрать строки матрицы в цикле `for`.
2. Взять каждую строку (Ряд). `currentRow` станет одномерным массивом `["Машина", 500000, "Игорь"]`.
3. Достать Цену (Индекс `1`, так как структура: 0 = Название, 1 = Цена, 2 = Менеджер).
4. Провести операцию ветвления (`if price > 10000`).
5. Если условие выполняется, пробросить `currentRow` в пустой резервуар-накопитель `cleanSales` (используя `.push()`).
6. После цикла очистить старый диапазон (`Clear`) и выполнить залповый `SetValue(cleanSales)`, динамически вычислив корректный размер адреса (например `"A2:C" + (1 + rows)`).

---

## 🧠 Глубинный разбор (Адресация Сборки)

Самая частая ошибка при выгрузке 2D-Массива: Несоответствие размерности (размера рамки).
Если `cleanSales` содержит 5 VIP-чеков, вы не можете выгрузить их в рамку большего размера типа `GetRange("A2:C100").SetValue(...)`. Остальные ячейки выдадут ошибку или забьются `undefined`.

Вы ДОЛЖНЫ гарантировать, что принимающий адрес с миллиметровой точностью соответствует длине сохраненного 2D-массива. 

**Формула размера:**
Мы начинаем выгрузку со 2 строки (т.к. 1-я — шапка).
Если размер массива 5 (`length === 5`), то последняя заполненная ячейка находится на строке 6 (строки: 2, 3, 4, 5, 6).
Поэтому вычисления рамки: `1 + 5 = 6`. Адрес: `"A2:C6"`.

## ✍️ Эталон алгоритма (Для самопроверки)

Вам понадобится выстроить следующую логику:

```javascript
for (let i = 0; i < matrix.length; i++) {
    let currentRow = matrix[i]; 
    let price = currentRow[1]; // Финансовая колонка
    
    // Предикат Фильтра
    if (price > 10000) {
        cleanSales.push(currentRow); // Сохраняем 1D-массив в новую 2D-матрицу
    }
}

if (cleanSales.length > 0) {
    // Точный расчет адреса рамки (Динамический Range)
    let addr = "A2:C" + (1 + cleanSales.length); 
    sheet.GetRange(addr).SetValue(cleanSales); // Массовая выгрузка
}
```
