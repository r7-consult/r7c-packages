# Интерактивный курс SmartDocs: «Разработка макросов с нуля»

## Цель курса
Довести пользователя от нулевого уровня до уверенного написания рабочих макросов для `cell`-редактора: запуск, отладка, автоматизация рутины и проверка результата через интерактивные карточки.

## Что берем из ресерча (и как применяем)
1. ONLYOFFICE docs по макросам: берем API-first подход, wrapper, цикл "написал -> запустил -> проверил".
2. Microsoft Office Scripts: берем паттерн "record -> edit", прогрессию `Workbook -> Worksheet -> Range`.
3. Google Apps Script fundamentals: берем последовательную структуру codelab-уроков от базы к проектам.
4. Coursera Excel/VBA specialization: берем проектную аттестацию и поэтапное усложнение.
5. ONLYOFFICE free macro course: берем короткие практические уроки и фокус на прикладных кейсах.

---

## Темы курса (сначала только темы)
1. Старт и базовый API.
2. JavaScript-основы для макросов.
3. Диапазоны, значения, формулы.
4. Листы и структура книги.
5. Работа с данными и проверка качества.
6. Форматирование и отчетный вид.
7. WorksheetFunction и custom functions.
8. Отладка, устойчивость, поддерживаемость.
9. Мини-проекты автоматизации.
10. Итоговый capstone.

## Уроки по темам + будущие тестовые задания

### 1) Старт и базовый API
1. Что такое макрос и где он исполняется.
Тест: объяснить ограничения среды и запустить пустой wrapper.
Проверка: `sheet_exists(Active)`.

2. Макроредактор SmartDocs: создать, сохранить, запустить.
Тест: создать макрос и записать маркер в `A1`.
Проверка: `cell_value(A1="ready")`.

3. Wrapper `(function () { ... })();` и строгий режим.
Тест: исправить код с необъявленной переменной.
Проверка: `cell_value(A2="ok")`.

4. `Api.GetActiveSheet()` и `GetRange()`.
Тест: получить активный лист и записать число в `B1`.
Проверка: `cell_value(B1=2, normalize:number)`.

5. `SetValue()` для значений и формул.
Тест: записать `B1=2`, `B2=2`, в `B3` формулу `=B1*B2`.
Проверка: `cell_formula(B3="=B1*B2")`, `cell_value(B3=4)`.

6. Пилотная карточка "Первые действия с Api.GetActiveSheet".
Тест: выполнить полное задание из карточки.
Проверка: пакет из 3 проверок (`sheet_exists`, `cell_formula`, `cell_value`).

### 2) JavaScript-основы для макросов
1. Переменные и типы в контексте таблиц.
Тест: записать строку в `A1` и число в `A2`.
Проверка: `cell_value(A1="text")`, `cell_value(A2=10, normalize:number)`.

2. Условия `if/else` для бизнес-логики.
Тест: при значении `B1>100` записывать статус `HIGH`, иначе `LOW`.
Проверка: `cell_value(C1="HIGH")`.

3. Циклы для диапазонов.
Тест: заполнить `A1:A10` числами `1..10`.
Проверка: `cell_value(A10=10, normalize:number)`.

4. Функции и декомпозиция.
Тест: вынести расчет в функцию `calcTotal`.
Проверка: `cell_value(D1=55, normalize:number)`.

5. Массивы и пакетная запись.
Тест: записать массив из 5 значений в строку `A1:E1`.
Проверка: `cell_value(E1=5, normalize:number)`.

6. Практика: "сумма четных".
Тест: вычислить сумму четных чисел в `A1:A10` и записать в `B1`.
Проверка: `cell_value(B1=30, normalize:number)`.

### 3) Диапазоны, значения, формулы
1. A1-нотация и адресация.
Тест: очистить `A1:C10`, заполнить `A1:C1` заголовками.
Проверка: `cell_value(B1="Qty")`.

2. Чтение и запись диапазонов.
Тест: считать `A2:A6`, вычислить сумму, записать в `A7`.
Проверка: `cell_value(A7=150, normalize:number)`.

3. Формулы в коде.
Тест: проставить в `D2:D6` формулу маржи.
Проверка: `cell_formula(D2="=C2-B2")`.

4. Ссылки относительные/абсолютные.
Тест: использовать `$B$1` в формуле скидки.
Проверка: `cell_formula(E2="=C2*$B$1")`.

5. Массовые операции.
Тест: скопировать формулу из `D2` до `D20`.
Проверка: `cell_formula(D20)` на ожидаемый шаблон.

6. Практика: мини-отчет продаж.
Тест: собрать таблицу и итоговую формулу `=SUM(D2:D6)`.
Проверка: `cell_formula(D7="=SUM(D2:D6)")`, `cell_value(D7)`.

### 4) Листы и структура книги
1. Активный лист vs лист по имени.
Тест: найти лист `Data`, записать метку в `A1`.
Проверка: `sheet_exists("Data")`, `cell_value("Data",A1="ok")`.

2. Создание листа.
Тест: создать лист `Report`.
Проверка: `sheet_exists("Report")`.

3. Переименование листа.
Тест: `Sheet2` переименовать в `Raw`.
Проверка: `sheet_exists("Raw")`.

4. Копирование данных между листами.
Тест: перенести итог из `Raw!B10` в `Report!B2`.
Проверка: `cell_value("Report",B2)`.

5. Защита от отсутствующих листов.
Тест: если листа нет, создать автоматически.
Проверка: `sheet_exists("Archive")`.

6. Практика: консолидация 3 листов.
Тест: собрать итоги в `Report!A1:B3`.
Проверка: `sheet_exists("Report")`, `cell_value("Report",B3)`.

### 5) Работа с данными и контроль качества
1. Подготовка данных: trim, пустые строки.
Тест: почистить `A2:A20`, записать счетчик очищенных в `F1`.
Проверка: `cell_value(F1, normalize:number)`.

2. Валидация чисел и дат.
Тест: проставить статус `INVALID` для невалидных строк.
Проверка: `cell_value(G2="INVALID")`.

3. Поиск дублей.
Тест: отметить дубли в `H` столбце.
Проверка: `cell_value(H5="DUP")`.

4. Статусы качества.
Тест: итоговый статус набора в `H1` (`OK`/`WARN`).
Проверка: `cell_value(H1="WARN")`.

5. Подготовка данных для отчета.
Тест: вывести набор "чистых" строк на лист `Clean`.
Проверка: `sheet_exists("Clean")`, `cell_value("Clean",A2)`.

6. Практика: quality-check pipeline.
Тест: полный проход очистки и маркировки.
Проверка: 6-8 комбинированных `cell_value` checks.

### 6) Форматирование и отчетный вид
1. Заголовки и базовый стиль.
Тест: применить стиль заголовка и поставить маркер в `Z1`.
Проверка (MVP): `cell_value(Z1="styled")`.

2. Формат чисел и процентов.
Тест: применить формат и записать контроль в `Z2`.
Проверка: `cell_value(Z2="fmt_ok")`.

3. Условная подсветка по порогу.
Тест: пометить убыточные строки и счетчик в `Z3`.
Проверка: `cell_value(Z3, normalize:number)`.

4. Структура финального отчета.
Тест: собрать блок KPI `A1:D6`.
Проверка: `cell_formula(D2)`, `cell_value(D6)`.

5. Повторное использование функций форматирования.
Тест: вынести `applyHeaderStyle(range)`.
Проверка: `cell_value(Z4="fn_ok")`.

6. Практика: monthly report layout.
Тест: оформить отчет и вернуть маркер завершения.
Проверка: `cell_value(Z5="done")`.

### 7) WorksheetFunction и custom functions
1. Введение в `ApiWorksheetFunction`.
Тест: вычислить сумму и среднее.
Проверка: `cell_value(B10)`, `cell_value(C10)`.

2. Статистические функции.
Тест: посчитать `MEDIAN` и `STDEV`.
Проверка: `cell_value(D10)`, `cell_value(E10)`.

3. Условные агрегаты.
Тест: `COUNTIF`/`SUMIF` для статуса.
Проверка: `cell_value(F10)`.

4. Функции дат.
Тест: вычислить возраст записи в днях.
Проверка: `cell_value(G10, normalize:number)`.

5. Custom function.
Тест: добавить пользовательскую функцию и применить в формуле.
Проверка: `cell_formula(H2)`, `cell_value(H2)`.

6. Практика: KPI-блок на функциях.
Тест: собрать KPI-строку из 5 показателей.
Проверка: 5 `cell_value` + 2 `cell_formula`.

### 8) Отладка и устойчивость
1. Логи и диагностические маркеры.
Тест: проставлять шаги выполнения в `Y1:Y5`.
Проверка: `cell_value(Y5="step5")`.

2. Запись макроса и рефакторинг.
Тест: взять recorded code и сократить дубли.
Проверка: `cell_value(Y6="refactored")`.

3. `try/catch` для ошибок.
Тест: перехватить ошибку и записать `Error: ...` в `A1`.
Проверка: `cell_value(A1)` по шаблону.

4. Защита от пустых диапазонов.
Тест: корректно завершать макрос при пустом вводе.
Проверка: `cell_value(Y7="empty_ok")`.

5. Идемпотентность запуска.
Тест: повторный запуск не ломает результаты.
Проверка: стабильный `cell_value` после двух запусков.

6. Практика: устойчивый производственный макрос.
Тест: пройти 2 сценария (валидный/ошибочный вход).
Проверка: пакет проверок + технический статус.

### 9) Мини-проекты автоматизации
1. Проект A: очистка и нормализация.
Тест: превратить "грязную" таблицу в чистую.
Проверка: 8-10 checks.

2. Проект B: консолидация регионов.
Тест: свести 3-5 листов в один отчет.
Проверка: `sheet_exists("Summary")`, формулы, итоги.

3. Проект C: план/факт.
Тест: рассчитать отклонения и статусы.
Проверка: `cell_formula`, `cell_value`.

4. Проект D: контроль качества.
Тест: отчет о пропусках/дублях.
Проверка: контрольные числа по каждой метрике.

5. Проект E: управленческий отчет.
Тест: собрать итоговый KPI-лист.
Проверка: multi-check (10+).

6. Защита мини-проектов.
Тест: стабилизировать код по фидбеку.
Проверка: повторный прогон без регрессии.

### 10) Capstone (итоговая аттестация)
1. Разбор ТЗ и входного workbook.
Тест: составить план реализации в комментариях.
Проверка: наличие обязательных шагов.

2. Реализация v1.
Тест: построить полный pipeline.
Проверка: минимум 10 автопроверок.

3. Исправление дефектов.
Тест: пройти набор негативных сценариев.
Проверка: отсутствие критических падений.

4. Оптимизация и упрощение.
Тест: сократить дубли в коде.
Проверка: маркер `refactor_done`.

5. Финальная сдача.
Тест: выполнить весь сценарий на чистом листе.
Проверка: 15-20 checks + ручной review.

6. Ретроспектива и дорожная карта роста.
Тест: добавить TODO-план улучшений.
Проверка: структурированный список next steps.

## Глобальные тестовые уровни (для генератора карточек)
1. `L1 Beginner`: 2-3 checks (`sheet_exists`, `cell_value`, `cell_formula`).
2. `L2 Basic+`: 4-6 checks + обработка edge cases.
3. `L3 Intermediate`: 6-10 checks + межлистовые сценарии.
4. `L4 Advanced`: 10-20 checks + ручная рубрика качества кода.

## Рубрика оценивания для будущего
1. Автопроверка (70%): структура книги 20%, формулы 25%, значения 25%.
2. Ручная оценка (30%): читаемость 10%, устойчивость 10%, переиспользование 10%.

## Контентный backlog
1. 60 карточек уроков (`10 модулей x 6 уроков`) в `draft:true`.
2. 10 модульных тестов.
3. 8 проектных карточек (мини-проекты + capstone-подсценарии).
4. Ручной review, затем перевод выбранных карточек в `draft:false`.
