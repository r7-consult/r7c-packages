# Урок 29. Инженерия Интеграции: Разбор шаблона "Чтение локальных файлов" (CSV Парсинг)

Мы приближаемся к финишу курса. В прошлых уроках мы вскрывали циклы и механизмы "Матриц". Но самый частый и сложный бизнес-запрос звучит так: *"У меня есть программа (Например: 1С, Битрикс, SAP). Она не умеет выгружать файлы в формате Р7-Офис. Она выгружает текстовые файлы (CSV, TXT). Как мне написать макрос, который сам (по кнопке) заберет этот файл из папки Скачивания, прочитает его и красиво разложит по колонкам в таблице?"*

Для решения таких задач в документации Р7-Офис существует шаблон `"Доступ к локальному файлу"`. 
В этом огромном теоретическом уроке (бизнес-справочнике) мы разберем анатомию этого скрипта и то, как макросы Р7 преодолевают изоляцию браузера, общаясь с файловой системой Windows/Linux!

## 🏛️ 1. Шаблонный код чтения CSV (Анатомия)

Представим файл `C:\Report\sales.csv`. Внутри него обычный текст (чеки):
```
Машина;500000;Иван
Ноутбук;150000;Анна
Мышь;3000;Петр
```

Вот как выглядит сырой шаблон-парсер из интернета (или документации Р7), адаптированный под этот сценарий:

```javascript
(function() {
    let sheet = Api.GetActiveSheet();
    let rowStart = 2; // Строка для вывода
    
    // ЗОНА 1: ВЫЗОВ C++ ЯДРА (Десктоп API)
    // Объект AscCommon — это системный мост к вашей операционной системе!
    
    // Читаем весь файл в одну гигантскую текстовую строку.
    // Если файла нет - код может упасть, нужна проверка!
    let fileText = AscCommon.ReadFile("C:\\Report\\sales.csv"); 
    
    if (fileText === undefined || fileText === null) {
        console.log("Ошибка: Файл не найден!");
        return; // Экстренный обрыв функции
    }

    // ЗОНА 2: ПАРСИНГ ТЕКСТА (Разделяй и властвуй)
    
    // 2.1. Режем монолитный текст на Массив Строк (По скрытому символу переноса "\n")
    // "\n" — это то, что происходит, когда вы нажимаете клавишу Enter.
    let linesArray = fileText.split("\n"); 
    
    let matrix2D = []; // Наш пустой кузов грузовика (Модуль 4)

    // ЗОНА 3: ЦИКЛ ПРЕВРАЩЕНИЯ (Форматировщик)
    for (let i = 0; i < linesArray.length; i++) {
        let currentString = linesArray[i]; // Пример: "Машина;500000;Иван"
        
        // Фильтр от пустых строк (Если в конце файла пустое нажатие Enter)
        if (currentString !== "") {
            
            // 3.1. Разрезаем конкретную строку по разделителю (Символ ;)
            // Метод split выдаст Массив [ "Машина", "500000", "Иван" ]
            let columns = currentString.split(";"); 
            
            // 3.2. Кладем этот Массивколонок в Кузов (формируя Двумерную Матрицу!)
            matrix2D.push(columns);
        }
    }
    
    // ЗОНА 4: ПОСЛЕДСТВИЯ ПОСКА (BATCH WRITE)
    // Кузов собран! Если там хоть что-то есть - Выплевываем Залпом!
    if (matrix2D.length > 0) {
        let widthStr = "C"; // У нас 3 колонки (A, B, C)
        let addr = "A" + rowStart + ":" + widthStr + (rowStart + matrix2D.length - 1);
        
        sheet.GetRange(addr).SetValue(matrix2D);
        console.log("Загружено " + matrix2D.length + " строк!");
    }
    
})();
```

## 🔬 2. Вскрытие и Реверс-Инжиниринг

Давайте разломаем эту сложную структуру на логические кирпичи. Почему этот шаблон такой мощный, и как он работает под капотом?

### 2.1 Изоляция Десктопа (Объект `AscCommon` / Builder)
Вы должны понимать главное архитектурное отличие. Если вы откроете Р7-Офис в браузере (Chrome / Версия для Сервера), этот скрипт **не сработает**. Браузеры из-за политик безопасности запрещают макросам лазать по жесткому диску пользователя `С:\`.
Но если пользователь открывает файл в **Настольном приложении (Десктопе) Р7**, движок C++ предоставляет макросам объект мост `AscCommon` (или класс `builder` в авто-скриптах). 
Команда `.ReadFile(путь)` позволяет мгновенно всосать гигабайты текста с флешки или сетевого диска прямо в оперативную память макроса!

*Обратите внимание: пути в Windows используют слэш `\`. А в синтаксисе JavaScript это служебный символ экранирования. Поэтому в пути к файлу мы пишем ДВОЙНОЙ слеш: `"C:\\Report\\sales.csv"`.*

### 2.2 Парсинг `text.split()`: Укрощение хаоса
Самый важный метод этого урока, без которого не существует современного программирования — это метод `.split(разделитель)`.
Функция `ReadFile` вернула нам одну монолитную колбасу (сплошной комбин текста). Р7 не умеет вставлять сплошной текст в таблицу (он запихнет всё в ячейку A1).

Мы используем `.split("\n")`.
`\n` — это невидимый символ новой строки (Line Feed). Метод `split` бежит по строке, и каждый раз, встречая `\n`, отрезает кусок мяса и кладет в массив `linesArray`.
Монолит превратился в список одномерных строк:
`linesArray[0]` = `"Машина;500000;Иван"`

Но это все еще единая длинная фраза! Нам нужны колонки (отдельная ячейка для 500 тыс)!
Поэтому **внутри цикла `(Зона 3)`** мы проводим Вторую Хирургию (Двойной Сплит).
Мы берем текущую моно-фразу `"Машина;500000;Иван"` и вызываем `.split(";")`. Мы приказываем: "Разорви фразу там, где видишь Точку-с-Запятой!".
Результат: Возвращается **Массив из 3 элементов**: `[ "Машина", "500000", "Иван" ]`.
А это (Внимание!) — именно та идеальная структура Ряда (Строки), которую мы ждем для складирования в кузов `matrix2D` и последующей залповой выгрузки `SetValue(2D)` (Модуль 4)!

### 2.3 Фильтр пустых обрывков (`!== ""`)
Почему в шаблоне установлена проверка `if (currentString !== "")`?
Очень часто корпоративные системы (1С/Excel) при выгрузке текстовых файлов оставляют в самом конце "Пустой Энтер" — лишнюю пустую строку.
Если скрипт попытается разрезать пустоту `("".split(";"))` и запихнуть в матрицу, он создаст кривой массив `[""]`. В момент залпа `SetValue` таблица Р7-Офис получит искаженные "грязные" данные.
Хороший архитектор (автор шаблона) предусмотрел это: он отсеивает абсолютно пустые строки. Система проглатывает только полезную нагрузку (Payload).

## ⚡ 3. Модификация под себя: Как улучшить этот шаблон?

Каждый шаблон из интернета или документации можно заточить под задачи вашей бухгалтерии. Что если разделитель у 1С не Точка-с-запятой (`;`), а Запятая (`,`) или Табуляция (`\t`)? 
Все очень просто!
Вам достаточно найти строку в Ядре:
`let columns = currentString.split(";");`

И поменять символ разделителя:
- Для Запятой (CSV-западный стандарт): `split(",")`
- Для Табуляции (TXT файлы): `split("\t")`
- Для Вертикальной черты (Unix-системы): `split("|")`

**Дополнительная модификация: "Налоги на лету"**
Раз уж вы прогоняете данные в цикле через матрицу, вы можете сразу применять математику "на лету" (до того, как вывалить их на лист).
Например, из 1С выгружается цена без НДС.
Вам нужна с НДС (20%).
Внутри `(Зона 3)` перед помещением `columns` в кузов, вы можете обратиться к ячейке с деньгами `columns[1]` (Индекс 1!):
```javascript
// Конвертируем текст "500000" в число и умножаем
let price = Number(columns[1]); 
let priceWithNds = price * 1.20;

// Перезаписываем в массиве ячейку обратно (с округлением)
columns[1] = Math.floor(priceWithNds);
// И уже потом отправляем в push!
```
Вот так интеграционный макрос Р7-Офис превращается в полноценную **ETL-машину** (Извлечение, Трансформация, Загрузка Данных).
