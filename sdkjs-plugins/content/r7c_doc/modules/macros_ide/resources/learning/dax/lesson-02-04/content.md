# Урок 7. Итераторы семейства X

Вступление

Представьте бухгалтера, который считает премии сотрудникам. Простая сумма зарплат не поможет — нужно для каждого сотрудника умножить зарплату на процент премии, а потом сложить результаты. Обычный SUM здесь бессилен, ведь он не может выполнить вычисление для каждой строки перед суммированием. Именно для таких задач в DAX существуют итераторы — функции семейства X, которые сначала вычисляют выражение для каждой строки, а затем агрегируют результаты.

Итераторы — это мост между простыми агрегатами и сложными вычислениями. Они раскрывают истинную мощь DAX, позволяя создавать метрики, невозможные в обычном Excel без вспомогательных столбцов. SUMX, AVERAGEX, MINX и MAXX дают гибкость построчных вычислений с эффективностью агрегатных функций. Без итераторов невозможно рассчитать взвешенные средние, суммы произведений или найти максимум вычисляемого выражения.

В предыдущем уроке мы изучили базовые агрегаты, которые работают с готовыми значениями столбцов. Теперь добавим возможность вычислять новые значения для каждой строки перед агрегацией.

## Что вы освоите в этом уроке

- Поймете концепцию итерации и строкового контекста в DAX
- Научитесь использовать SUMX для сумм произведений и взвешенных расчетов
- Освоите AVERAGEX для корректных взвешенных средних
- Изучите MINX и MAXX для поиска экстремумов вычисляемых выражений

В следующем уроке мы перейдем к логическим функциям и обработке пропусков, что позволит создавать надежные меры с проверками условий.

Теоретическая часть

Основные понятия

Итератор — это функция DAX, которая выполняет два действия: сначала проходит по каждой строке таблицы (итерация), вычисляя заданное выражение, затем агрегирует полученные результаты. Название функций-итераторов заканчивается на X: SUMX, AVERAGEX, MINX, MAXX, что указывает на их способность выполнять выражение (eXpression) для каждой строки. В отличие от простых агрегатов, которые работают только с существующими столбцами, итераторы могут вычислять сложные выражения “на лету”.

Строковый контекст (Row Context) — это специальный контекст вычисления, который создается итератором для каждой строки обрабатываемой таблицы. В строковом контексте доступны значения всех столбцов текущей строки. Представьте, что итератор — это палец, который движется по таблице строка за строкой, и в каждый момент времени “видит” только одну текущую строку. Это позволяет создавать выражения, использующие значения из разных столбцов одной строки.

Выражение итератора — это формула, которая вычисляется для каждой строки в строковом контексте. Выражение может быть простым (ссылка на столбец) или сложным (произведение нескольких столбцов, условные вычисления). Например, в SUMX(Sales, Sales[Quantity] * Sales[Price]) выражение Sales[Quantity] * Sales[Price] вычисляется для каждой строки таблицы Sales.

Таблица итерации — первый параметр любой функции-итератора, определяющий набор строк для обработки. Это может быть физическая таблица модели, результат табличной функции или фильтрованная таблица. Размер и состав этой таблицы напрямую влияет на результат итератора и производительность вычисления.

Ключевая механика

Механизм работы итераторов можно представить как конвейер с двумя этапами. На первом этапе итератор получает таблицу и начинает обход строка за строкой. Для каждой строки создается строковый контекст, в котором вычисляется указанное выражение. Результаты вычислений временно сохраняются в виртуальном столбце. На втором этапе к этому виртуальному столбцу применяется соответствующая агрегатная операция: сумма для SUMX, среднее для AVERAGEX, минимум для MINX, максимум для MAXX.

Представьте итератор как умного помощника с калькулятором и блокнотом. Вы даете ему таблицу и формулу. Помощник берет первую строку, подставляет значения из строки в формулу, вычисляет результат и записывает в блокнот. Затем переходит ко второй строке и повторяет процесс. После обработки всех строк помощник выполняет финальное действие с записанными результатами — складывает их, находит среднее или определяет экстремум.

Критическое отличие от вычисляемых столбцов заключается в том, что итераторы не создают постоянных данных в модели. Виртуальный столбец с результатами существует только во время вычисления меры и исчезает сразу после получения агрегированного результата. Это экономит память и позволяет создавать сложные вычисления без раздувания модели данных.

Взаимодействие с фильтровым контекстом происходит до начала итерации. Сначала к исходной таблице применяются все активные фильтры из отчета, создавая отфильтрованную таблицу. Затем итератор работает только с видимыми строками. Это означает, что один и тот же итератор будет давать разные результаты в зависимости от применных фильтров, автоматически адаптируясь к контексту использования.

Производительность итераторов зависит от двух факторов: размера итерируемой таблицы и сложности выражения. Простые выражения (умножение двух столбцов) выполняются быстро даже на миллионах строк. Сложные выражения с вложенными условиями могут существенно замедлить вычисления. Важно помнить: итератор выполняет выражение для каждой видимой строки, поэтому оптимизация фильтров критична для производительности.

Синтаксис и правила

Базовый синтаксис итераторов: ФУНКЦИЯX(Таблица, Выражение). Первый параметр — всегда таблица или табличное выражение. Второй параметр — выражение, вычисляемое для каждой строки. В выражении можно использовать столбцы из итерируемой таблицы без указания имени таблицы, так как контекст однозначен: SUMX(FactInternetSales, [OrderQuantity] * [UnitPrice]).

```dax
SUMX суммирует результаты выражения по всем строкам. Типичное применение — расчет суммы, когда нужно перемножить значения из разных столбцов. Синтаксис: SUMX(Table, Expression). Если выражение возвращает BLANK() для какой-то строки, эта строка игнорируется в сумме. Пример: SUMX(FactInternetSales, FactInternetSales[OrderQuantity] * FactInternetSales[UnitPrice]).
AVERAGEX вычисляет среднее арифметическое результатов выражения. Важно: среднее вычисляется по количеству строк с не-BLANK результатами. Это позволяет корректно рассчитывать взвешенные средние. Синтаксис идентичен SUMX: AVERAGEX(Table, Expression). Классическое применение — средняя цена с учетом количества.
```

MINX и MAXX находят минимальное и максимальное значение среди результатов выражения. Работают не только с числами, но и с датами и текстом. Полезны для поиска экстремумов вычисляемых значений, например, минимальной прибыльности или максимальной скидки. Синтаксис аналогичен другим итераторам: MINX(Table, Expression) и MAXX(Table, Expression).

Сравнение с альтернативами

В Excel для аналогичных расчетов потребовались бы вспомогательные столбцы. Чтобы посчитать сумму произведений количества на цену, пришлось бы создать столбец с формулой =Количество*Цена, а затем суммировать его. В DAX итератор SUMX выполняет оба действия в одной формуле, не создавая промежуточных данных. Это делает модель компактнее и быстрее.

Вычисляемые столбцы в DAX могли бы дать похожий результат — создать столбец Revenue = [Quantity] * [Price], затем использовать SUM([Revenue]). Но это увеличит размер модели и время обновления. Итераторы выполняют вычисления только когда нужно и только для видимых данных, что эффективнее при редком использовании метрики.

SQL достигает похожего результата через подзапросы или вычисляемые поля в SELECT. Но в SQL вычисление происходит при выполнении запроса на сервере базы данных. Итераторы DAX работают с данными, уже загруженными в память, что обеспечивает мгновенный отклик при изменении фильтров в интерактивных отчетах.

Частые ошибки

Использование итератора там, где достаточно простого агрегата — распространенная ошибка. Если нужно просто просуммировать столбец, используйте SUM, а не SUMX. Итераторы медленнее простых агрегатов из-за необходимости создания строкового контекста. Применяйте итераторы только когда нужны вычисления на уровне строк.

Неправильное понимание области итерации приводит к неверным результатам. Итератор работает только с теми строками, которые видны в текущем фильтровом контексте. Если в отчете выбран 2014 год, SUMX обработает только строки 2014 года. Это особенность, а не ограничение, но требует понимания при создании мер.

Забывание о BLANK в выражениях итераторов создает неожиданные результаты. Если выражение содержит умножение, и один из операндов BLANK, весь результат будет BLANK, и строка не учтется в агрегации. Это может исказить средние значения или суммы. Всегда проверяйте обработку пустых значений в сложных выражениях.

Создание слишком сложных выражений в итераторах ухудшает читаемость и производительность. Если выражение занимает несколько строк с множественными условиями, лучше разбить его на несколько мер. Это улучшит поддерживаемость кода и может повысить производительность за счет кэширования промежуточных результатов.

Ответы

SUMX сначала вычисляет выражение для каждой строки, затем суммирует результаты; SUM просто суммирует готовые значения столбца.

Строковый контекст создается итератором для каждой строки таблицы, давая доступ ко всем столбцам текущей строки.

AVERAGEX принимает два параметра: таблицу для итерации и выражение, вычисляемое для каждой строки.

Да, MINX работает с текстом, возвращая первое значение в алфавитном порядке среди результатов выражения.

Итераторы создают строковый контекст для каждой строки, что требует больше ресурсов, чем простое суммирование столбца.

Если выражение возвращает BLANK для строки, эта строка игнорируется в финальной агрегации.

SUMX эффективнее когда вычисление используется редко, так как не занимает место в модели и вычисляется только при необходимости.

Заключение

В этом уроке мы освоили мощнейший инструмент DAX — итераторы семейства X. Вы научились использовать SUMX для динамических вычислений сумм произведений, AVERAGEX для корректных взвешенных средних, MINX и MAXX для поиска экстремумов в вычисляемых выражениях. Понимание строкового контекста открывает путь к созданию сложных метрик без загромождения модели вычисляемыми столбцами.

Практическое применение итераторов огромно: расчет выручки с учетом скидок, взвешенные средние цены, анализ прибыльности на уровне транзакций.
