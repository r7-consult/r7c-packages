# Урок 30. Финальная Практика Аудитора: Архитектура Мутации и Выгрузки

Вы изучили вычисления, операции форматирования и механизмы Batch Write для матриц данных (Модуль 4 и 5). Задача финальной практики объединить все эти концепции в один пайплайн. Это классическое "тестовое задание" на вакансию разработчика Р7-Офис или 1С-Аналитика. 

Больше не будет подробных подсказок. Вы должны спроектировать алгоритм самостоятельно.

## 🎯 Бизнес-задача: Скрипт-Аудитор

Имеется таблица с массивом исходных транзакций (Колонка А: Артикул товара, B: Доход с партии, C: Себестоимость). Всего 20 тысяч строк. Требуется реализовать 3 бизнес-правила (Business Rules):

1. **Трансформация (ETL):** Вычислить Чистую Прибыль (ЧП = Доход - Расход) математически в памяти JS (Не используя формулы Excel).
2. **Мутация UI:** Произвести цветовое форматирование исходного листа `Sheet1` (Highlight). Если партия прибыльная (ЧП > 0), ячейка артикула (столбец A) должна стать Зеленой. Иначе — Красной.
3. **Фильтрация и Маршрутизация:** Если `ЧП > 50 000` рублей, партия классифицируется как VIP. Скрипт должен составить отдельный 2D Array из VIP-строк. По завершению цикла, скрипт проверяет наличие в книге вкладки `VIP_Сделки` (создаёт её, если не существует), очищает полигон, производит динамический расчет каретки (длины массива) и выгружает `superDeals` методом `SetValue`.

---

## 📝 Разбор Алгоритма (Архитектурные требования)

**1. Индексы Матрицы:**
Строка сделки `row = matrix[i]`. Доход имеет индекс `1`, Себестоимость `2`.

**2. Управление Графикой:**
Для раскраски ячейки Артикула вам потребуется обращение к интерфейсу Р7: `sheet.GetRange(...)` и метод `SetFillColor()`. *Обратите внимание на смещение координат ячейки! Матрица извлекается из `A2`, следовательно `i` (0) равен физической строке `2`. Адрес ячейки – `"A" + (i + 2)`.*

**3. Фабрика Листов:**
При работе с `VIP_Сделки` вы используете уже знакомый вам паттерн Factory Method: `Api.GetSheet("Имя")`. Вы проверяете его на `null` и используете `Api.AddSheet()`. Затем вам необходимо вычислить размер матричного адреса выгрузки `"A1:C" + MathRows` и сбросить матрицу.

---

## 🧠 Глубинный анализ: Смешивание UI и RAM

В Enterprise-архитектуре программисты избегают смешивания операций в Оперативной Памяти (Матрицы ОЗУ) и интерфейсных вызовов (UI DOM Manipulation). 
Причина: Вызов `SetFillColor` внутри JS-цикла на 50 000 итераций вызывает лаги (Performance Hit). Идеальный скрипт производит вычисления за миллисекунды, выгружает данные через Batch Write, и только потом применяет к листу Conditional Formatting (Условное Форматирование) или сторонние плагины.

Однако, мы требуем использовать `SetFillColor` внутри цикла, потому что этот анти-паттерн является стандартом "де-факто" в мелких офисных макросах на базах до 5 000 строк (Покрасить красным неоплаченное). Вы обязаны уметь быстро реализовывать эту механику.

## ✍️ Референсная структура (Code Snippet)

Для самопроверки сверьтесь с данной топологией кода:

```javascript
/* === БЛОК 1: ETL И ГРАФИКА === */
for (let i = 0; i < matrix.length; i++) {
    let row = matrix[i];
    let profit = row[1] - row[2]; // Math
    
    // Смещение DOM (DOM Offset)
    let cellAddress = "A" + (i + 2);
    
    // Предикаты (If...Else)
    if (profit > 0) {
        sheet.GetRange(cellAddress).SetFillColor(colorGreen);
        
        // Push Into RAM Array
        if (profit > 50000) superDeals.push(row);
    } else {
        sheet.GetRange(cellAddress).SetFillColor(colorRed);
    }
}

/* === БЛОК 2: ФАБРИКА И БЭТЧИНГ === */
if (superDeals.length > 0) {
    let vipSheet = Api.GetSheet("VIP_Сделки");
    
    // Singleton Factory
    if (vipSheet === null) vipSheet = Api.AddSheet("VIP_Сделки");
    
    vipSheet.GetRange("A1:C100").Clear();
    
    // Batch Write Allocation
    let addr = "A1:C" + superDeals.length;
    vipSheet.GetRange(addr).SetValue(superDeals);
}
```
