# Введение в макросы Р7-Офис и основы JavaScript

Добро пожаловать в курс **«Разработка макросов с нуля до PRO»**! 

В этом первом уроке мы заложим фундамент. Макросы в Р7-Офис пишутся на современном языке программирования — **JavaScript** (в отличие от VBA в Excel). Это дает огромные преимущества: язык мощный, популярный и позволяет не только работать с таблицами, но и интегрироваться с веб-сервисами.

## 1. Что такое макрос?
**Макрос** — это скрипт (небольшая программа), который автоматизирует рутинные действия в таблице. Например, макрос может:
- Пройти по всем строкам и удалить дубликаты.
- Найти ячейки с определенными значениями и закрасить их красным.
- Собрать данные с 10 разных листов и свести их в один итоговый отчет.

## 2. Базовый синтаксис JavaScript
Поскольку мы пишем на JS, вам нужно знать несколько главных правил:

### Переменные (Где хранятся данные)
Для создания переменной используется ключевое слово `let` (значение может меняться) или `const` (значение неизменно).
```javascript
let myName = "Иван"; // Строка
const age = 30;      // Число
let isActive = true; // Логическое значение (Да/Нет)
```

### Вывод сообщений
Хотя в макросах мы обычно меняем ячейки, иногда полезно вывести сообщение для отладки. Для этого используется `console.log()`:
```javascript
console.log("Привет, мир!");
```

### Условия (Если... то...)
Как научить программу принимать решения? Использовать `if` и `else`.
```javascript
let score = 85;
if (score > 80) {
    console.log("Отлично!");
} else {
    console.log("Нужно еще постараться.");
}
```

### Циклы (Повторение действий)
Если нужно выполнить действие много раз (например, перебрать 100 строк), используют циклы, например `for` или `while`.
```javascript
for (let i = 0; i < 5; i++) {
    console.log("Это шаг номер: " + i);
}
```

## 3. Архитектура Эталонного Макроса Р7-Офис (Best Practices)
Каждый профессиональный макрос Р7-Офис, как показывают официальные шаблоны разработчиков (например, `TargetMacro.js`), всегда обернут в жесткую защитную оболочку. Мы сразу будем учиться писать код как `Senior Developer`.

Забудьте про скрипты из 2 строчек. Правильный макрос всегда выглядит так:

```javascript
(function() {
    'use strict'; // 1. Строгий режим (Запрещает "грязный" код)
    
    try {
        // 2. Блок попытки выполнить код
        const api = Api;
        if (!api) throw new Error('API недоступен');
        
        // --- ЗДЕСЬ ПИШЕТСЯ ЯДРО ВАШЕГО МАКРОСА ---
        console.log("Привет, мир!");
        // ----------------------------------------
        
        console.log('Macro executed successfully'); // Успех
        
    } catch (error) {
        // 3. Блок перехвата Ошибок (Если код выше упал)
        console.error('Macro execution failed:', error.message);
        // Безопасный вывод текста ошибки прямо в ячейку A1
        if (typeof Api !== 'undefined' && Api.GetActiveSheet) {
            let sheet = Api.GetActiveSheet();
            if (sheet) sheet.GetRange('A1').SetValue('Ошибка: ' + error.message);
        }
    }
})();
```

**Зачем эта "броня"?** 
1. `(function(){})();` — Изолирует память. Ваши переменные не сломают чужие плагины.
2. `'use strict'` — Блокирует опечатки. Без него браузер прощает ошибки, с ним — требует идеальной дикции.
3. `try / catch` — Главное отличие любителя от инженера. Если макрос упадет, программа не "зависнет" интерфейсом. Ошибка элегантно "поймается" в блок `catch` и будет выведена пользователю красным текстом в ячейку A1!
