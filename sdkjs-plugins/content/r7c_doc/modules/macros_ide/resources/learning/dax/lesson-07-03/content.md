# Урок 26. Динамические метки и текстовые итоги

Вступление

Представьте презентацию для совета директоров, где вместо безликого заголовка “Продажи: $5,234,567” появляется динамическая подпись: “Продажи велосипедов в Северной Америке за Q3 2013: $5,234,567 (+23% к прошлому году)”. Каждый элемент этого заголовка меняется автоматически при выборе фильтров, создавая контекстно-зависимое повествование. В DAX функции SELECTEDVALUE и CONCATENATEX превращают статичные отчеты в интерактивные истории, где каждая цифра сопровождается понятным описанием.

Динамические метки критически важны для самодокументируемых отчетов. Пользователь не должен гадать, какие фильтры применены и что означают числа. Текстовые итоги объединяют несколько метрик в единое информативное сообщение. Это особенно ценно для руководителей, которым нужны готовые выводы, а не сырые данные. DAX позволяет создавать интеллектуальные текстовые элементы, адаптирующиеся к любому контексту анализа.

В предыдущем уроке мы изучили сегментацию и бинирование. Теперь научимся создавать динамические описания для этих сегментов и других элементов отчетов.

Теоретическая часть

Основные понятия

```dax
SELECTEDVALUE — функция, возвращающая значение столбца, если в текущем контексте выбрано ровно одно значение. SELECTEDVALUE(‘DimProduct’[EnglishProductName]) вернет название продукта, если выбран один продукт, или BLANK (или альтернативное значение), если выбрано несколько или ни одного. Это основа для создания динамических заголовков, реагирующих на выбор пользователя. SELECTEDVALUE идеально подходит для захвата единичных выборов в срезах и фильтрах.
CONCATENATEX — функция объединения текстовых значений из таблицы в единую строку. CONCATENATEX(VALUES(‘DimProduct’[Color]), ‘DimProduct’[Color], ", ") создаст строку со всеми выбранными цветами через запятую: “Red, Blue, Black”. Функция итерирует по таблице, извлекает текстовые значения и склеивает их через указанный разделитель. Это позволяет создавать списки выбранных элементов для отображения в заголовках.
```

Динамическая метка — это текстовый элемент отчета, автоматически изменяющийся в зависимости от контекста. Вместо статичного “Продажи” метка может показывать “Продажи велосипедов за 2013 год в США” или “Общие продажи всех товаров”, адаптируясь к примененным фильтрам. Динамические метки делают отчеты самоочевидными и снижают вероятность неправильной интерпретации.

Текстовый итог — это мера, возвращающая не число, а форматированную текстовую строку с несколькими показателями. Например: “Всего продано 1,234 единиц на сумму $567,890 (средний чек: $461)”. Такие итоги объединяют ключевые метрики в одном месте, создавая готовые выводы для пользователя.

Ключевая механика

Механизм работы SELECTEDVALUE основан на проверке количества уникальных значений в контексте. Функция внутренне выполняет HASONEVALUE для проверки единственности, затем возвращает это значение через VALUES или MIN/MAX. Если значений несколько или ноль, возвращается второй параметр (альтернативное значение) или BLANK. Это обеспечивает безопасный захват выборок без ошибок.

Представьте SELECTEDVALUE как умного секретаря, который слушает совещание. Если все говорят об одном продукте, секретарь записывает его название. Если обсуждают несколько продуктов или вообще не упоминают продукты, секретарь пишет “Несколько” или “Все продукты”. Это гарантирует, что заголовок всегда будет информативным и корректным.

CONCATENATEX работает как сборочная линия для текста. Каждая строка таблицы проходит через конвейер, из нее извлекается текстовое значение, добавляется разделитель, и результат присоединяется к общей строке. Важно: CONCATENATEX сохраняет порядок сортировки таблицы, что позволяет контролировать последовательность элементов в результирующей строке.

Комбинирование функций создает интеллектуальные метки. SELECTEDVALUE захватывает единичные выборы, CONCATENATEX — множественные. Условная логика через IF определяет, какую функцию использовать. Результат — метка, которая всегда точно описывает текущий контекст, независимо от количества выбранных элементов.

Форматирование и локализация важны для профессиональных меток. Числа должны быть отформатированы с разделителями тысяч, даты — в привычном формате, проценты — с нужным количеством знаков. DAX предоставляет функции FORMAT и UNICODE для создания красивых и понятных текстовых представлений.

```dax
SELECTEDVALUE имеет два параметра: SELECTEDVALUE(columnName, [alternateResult]). Первый — столбец для проверки, второй (опциональный) — значение при множественном или пустом выборе:
```

```dax
[Dynamic Title] :=
```

Сравнение с альтернативами

В Excel для динамических заголовков используются формулы СЦЕПИТЬ или ТЕКСТ с ссылками на ячейки. Но они не адаптируются автоматически к изменению фильтров и требуют ручного обновления. DAX-метки обновляются мгновенно при любом взаимодействии с отчетом.

SQL не предназначен для создания динамических меток в интерактивных отчетах. SQL может генерировать статичные заголовки при формировании запроса, но не может реагировать на действия пользователя. DAX работает в реальном времени, создавая контекстно-зависимые описания.

JavaScript в веб-дашбордах предлагает полную гибкость для динамических элементов, но требует программирования. DAX проще для бизнес-пользователей и интегрирован с инструментами визуализации, не требуя написания кода обработки событий.

Частые ошибки

Использование SELECTEDVALUE без альтернативного значения может привести к пустым заголовкам при множественном выборе. Всегда указывайте второй параметр с осмысленным текстом типа “Все”, “Несколько” или “Не выбрано”. Это гарантирует информативность метки в любой ситуации.

Игнорирование сортировки в CONCATENATEX создает непредсказуемый порядок элементов. Без явной сортировки порядок зависит от внутренней структуры данных и может меняться. Всегда указывайте параметры orderBy и order для консистентного результата.

Создание слишком длинных конкатенированных строк ухудшает читаемость. Если выбрано 50 продуктов, список через запятую займет несколько строк. Ограничивайте длину через TOPN или показывайте количество вместо списка при большом выборе.

Отсутствие форматирования чисел в текстовых итогах снижает профессионализм отчетов. Число 1234567.89 трудно читать. Используйте FORMAT для создания “1,234,567.89” или “1.23M”. Правильное форматирование критично для восприятия информации.

## DAX-решение

```dax
// Форматированные продажи
[Sales Formatted] :=
FORMAT(SUM(FactInternetSales[SalesAmount]), "$#,##0")
// Количество заказов
[Orders Count] :=
DISTINCTCOUNT(FactInternetSales[SalesOrderNumber])
// Текстовый итог
[Executive Summary] :=
"Total Sales: " & [Sales Formatted] &
" | Orders: " & FORMAT([Orders Count], "#,##0") &
" | Avg Order: " & FORMAT(DIVIDE(SUM(FactInternetSales[SalesAmount]), [Orders Count]), "$#,##0")
```

Проверка результата: Общий итог: “Total Sales: $29,358,677 | Orders: 27,659 | Avg Order: $1,062” За 2013: “Total Sales: $9,791,060 | Orders: 9,328 | Avg Order: $1,050” Bikes: “Total Sales: $28,318,145 | Orders: 13,339 | Avg Order: $2,123” Итог корректно агрегирует и форматирует все показатели

Ответы

```dax

CONCATENATEX принимает таблицу, выражение для извлечения, разделитель (опционально), поле и порядок сортировки.
```

Альтернативный результат обеспечивает информативность метки при множественном или пустом выборе.

Через параметры orderBy (поле сортировки) и order (ASC/DESC) можно задать порядок элементов.

Да, SELECTEDVALUE работает с любыми типами данных, включая числа и даты.

CONCATENATEX возвращает пустую строку при пустой входной таблице.

Через оператор конкатенации &amp; можно объединять результаты нескольких динамических функций.
