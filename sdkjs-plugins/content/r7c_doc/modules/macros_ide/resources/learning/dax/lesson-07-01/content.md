# Урок 24. Доля в общем и ранжирование

Вступление

Представьте совет директоров крупной компании, где каждый департамент борется за бюджет. Просто сказать “наш отдел принес 5 миллионов” недостаточно — важно показать, что это 40% от общей прибыли компании и первое место среди всех подразделений. Доли и рейтинги превращают абсолютные числа в контекстуальные инсайты. В DAX функции RANKX и TOPN, комбинированные с ALL и REMOVEFILTERS, создают динамические рейтинги и процентные показатели, адаптирующиеся к любому уровню анализа.

Расчет долей и ранжирование — это основа конкурентного анализа и стратегического планирования. Топ-10 клиентов, доля рынка по регионам, ранг продукта в категории — эти метрики определяют бизнес-решения. Без специализированных функций пришлось бы вручную сортировать данные и пересчитывать проценты при каждом изменении фильтров. DAX автоматизирует эти расчеты, делая их динамичными и контекстно-зависимыми.

Теоретическая часть

Основные понятия

```dax
RANKX — функция ранжирования, присваивающая порядковый номер каждому элементу на основе значения выражения. RANKX(ALL(‘DimProduct’), [Sales Amount]) присвоит ранг 1 продукту с максимальными продажами среди всех продуктов. Функция работает в текущем контексте, автоматически пересчитывая ранги при изменении фильтров. RANKX может обрабатывать одинаковые значения, присваивая им одинаковые ранги или пропуская следующие позиции.
```

TOPN — функция выделения N верхних или нижних элементов по указанному критерию. TOPN(10, ALL(‘DimProduct’), [Sales Amount]) вернет таблицу с 10 продуктами с максимальными продажами. В отличие от простой сортировки, TOPN создает динамическую таблицу, которую можно использовать в CALCULATE для фильтрации. Это позволяет рассчитывать метрики только для топ-элементов.

Доля от общего (Percentage of Total) — это отношение части к целому, выраженное в процентах. В DAX это обычно реализуется через деление текущего значения на значение с удаленными фильтрами: [Sales Amount] / CALCULATE([Sales Amount], ALL(‘DimProduct’)). Ключевой момент — правильное определение “общего”: это может быть общий итог по всем данным, по категории, по периоду.

Контекстная адаптация рангов означает, что ранги пересчитываются в зависимости от примененных фильтров. Если выбрана категория “Bikes”, RANKX покажет ранги только среди велосипедов. Если добавить фильтр по региону, ранги пересчитаются для велосипедов в этом регионе. Это делает анализ гибким и многоуровневым.

Ключевая механика

Механизм работы RANKX основан на создании временной отсортированной таблицы и присвоении позиций. Функция проходит по каждой строке указанной таблицы, вычисляет выражение для ранжирования, сортирует результаты, присваивает ранги. Важно: RANKX выполняется для каждой строки визуализации, что может влиять на производительность при больших объемах данных.

Представьте RANKX как спортивное соревнование. Каждый участник (строка таблицы) показывает результат (значение выражения). Судья (RANKX) выстраивает участников по результатам и присваивает места. Если два участника показали одинаковый результат, они делят место. Контекст определяет, кто участвует в соревновании — все спортсмены или только из определенной категории.

Расчет долей требует двух вычислений: числителя (часть) и знаменателя (целое). Числитель обычно считается в текущем контексте, знаменатель — с удаленными определенными фильтрами. ALL удаляет все фильтры, ALLEXCEPT сохраняет выбранные, ALLSELECTED работает с видимой выборкой. Выбор функции определяет базу для сравнения.

TOPN работает как фильтр VIP-зоны. Из всех гостей (строк таблицы) выбираются N самых важных по критерию (выражению). Эти VIP-гости попадают в отдельную зону (виртуальную таблицу), где для них можно провести специальные расчеты. TOPN часто комбинируется с CALCULATE для вычисления метрик только для лидеров.

Обработка равных значений в ранжировании может происходить двумя способами. При Dense ранжировании одинаковые значения получают одинаковый ранг, следующий уникальный получает следующий номер (1,2,2,3). При Skip ранжировании после одинаковых рангов пропускаются позиции (1,2,2,4). RANKX поддерживает оба варианта через параметр ties.

## Для расчета долей используется паттерн с DIVIDE и ALL

```dax
[% of Total Sales] :=
DIVIDE(
SUM(FactInternetSales[SalesAmount]),
CALCULATE(SUM(FactInternetSales[SalesAmount]), ALL('DimProduct'))  // убираем фильтр продуктов
)
```

Сравнение с альтернативами

В Excel для ранжирования используется функция РАНГ или РАНГ.СР, но она статична и требует пересчета при изменении данных. Для процентов используются абсолютные ссылки ($), но они не адаптируются к фильтрам. DAX автоматически пересчитывает ранги и доли при любом изменении контекста.

SQL оконные функции ROW_NUMBER(), RANK(), DENSE_RANK() концептуально похожи на RANKX, но применяются при запросе. SQL-подход эффективен для статических отчетов, DAX — для интерактивной аналитики с постоянно меняющимися фильтрами.

Power Query может предрассчитать ранги при загрузке через Table.AddRankColumn, но это увеличивает объем данных и не адаптируется к фильтрам. Любое изменение в ранжировании требует перезагрузки. DAX вычисляет на лету без дублирования данных.

Частые ошибки

Использование RANKX без ALL приводит к постоянному рангу 1, так как в контексте каждой строки остается только она сама. Всегда включайте ALL или другую таблицу с полным набором элементов для корректного ранжирования.

Неправильный выбор базы для процентов искажает результаты. Если нужна доля от категории, но используется ALL без параметров, получится доля от всего. Тщательно определяйте, какие фильтры нужно удалить для правильного знаменателя.

Путаница между TOPN как фильтром и как таблицей создает ошибки. TOPN возвращает таблицу, которую нужно использовать в CALCULATE или итераторах. Попытка использовать TOPN как меру напрямую вызовет ошибку.

Игнорирование производительности при ранжировании больших таблиц замедляет отчеты. RANKX выполняется для каждой строки визуализации. При тысячах строк это может стать узким местом. Оптимизируйте через материализацию или ограничение области ранжирования.

Ответы

RANKX присваивает ранги элементам таблицы на основе выражения; принимает таблицу, выражение, значение, порядок и способ обработки связей.

TOPN создает динамическую таблицу с N элементами, которую можно использовать как фильтр; сортировка только упорядочивает визуализацию.

Через деление текущего значения на значение с удаленными фильтрами: [Measure] / CALCULATE([Measure], ALL(Table)).

ALL обеспечивает полный набор элементов для ранжирования; без него каждый элемент будет иметь ранг 1.

TOPN возвращает таблицу с N верхними или нижними элементами по указанному критерию.

Ранги автоматически пересчитываются в текущем контексте; при фильтре категории ранжирование происходит внутри категории.

Dense (плотное) — одинаковые значения получают одинаковый ранг; Skip — после одинаковых рангов позиции пропускаются.
