# AUTHORING GUIDE

## Минимальный чек-лист автора

1. Урок в формате Markdown + YAML frontmatter.
2. `interactive.enabled: true`, `lessonId`, `title`, `targetEditor: "cell"` заполнены.
3. В `task.goal` явно описан ожидаемый результат.
4. `starterCode` запускается без синтаксических ошибок.
5. `beforeScript` делает урок повторяемым (сброс диапазона/подготовка данных).
6. `checks` отражают критерии принятия (`sheet_exists`, `cell_value`, `cell_formula`).
7. Для новых карточек `draft: true` до ручной проверки.

## Ревью перед публикацией

1. Открыть урок в SmartDocs и выполнить его end-to-end.
2. Проверить поведение кнопок `Запуск`, `Проверить`, `Сброс`.
3. Проверить сообщения ошибок при намеренно неверном коде.
4. Проверить, что критерии из `task.acceptance` совпадают с `checks`.
5. Переключить `draft: false` только после успешного прогона.

## Политика черновиков

- `draft: true` скрыт из обычной навигации.
- Для dev-просмотра черновиков используйте `localStorage`-флаг:
  `sd.interactive.showDrafts = "1"`.
