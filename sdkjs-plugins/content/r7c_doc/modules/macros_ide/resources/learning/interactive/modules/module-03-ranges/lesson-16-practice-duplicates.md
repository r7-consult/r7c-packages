# Урок 16. Практика Инжиниринга: Механизм Маркировки Дубликатов (Caching)

В VBA задача поиска дубликатов часто решалась через коллекции, словари `Scripting.Dictionary` или прямым вложенным циклом сравнения. Метод "Remove Duplicates" интерфейса Excel физически уничтожал строки, а задача "Выделения дубликатов цветом" требовала макроса. 

В Р7-Офис (JavaScript V8) мы реализуем эту процедуру через кэширование в памяти: массив или `Set` (коллекцию уникальных значений).

## 🎯 Бизнес-задача: Аудит Реестра
Ваш реестр содержит уникальные ИНН (строки 1-10, колонка A). Из 10 000 строк часть — мусорные повторы, созданные операторами. Вы пишете скрипт, проходящий `O(N)` сканированием сверху вниз, записывая каждый считанный хэш-ключ `docNum` в локальный кэш-массив `seen`. Если `includes()` возвращает True (ИСТИНА) — значит, это дубликат. Скрипт помечает эту конкретную (повторную) ячейку цветом `yellow` для визуального аудита.

Первый экземпляр игнорируется, оставаясь белым (уникальным).

---

## 📝 Постановка задачи

В заготовке кода вы:
1. Создали инстанс массива `seen = []`.
2. Запустили Range-Цикл (1 to 10). Обратите внимание: метод `GetValue()` обернут в функцию форсированной типизации `String()`. В JS "123" строго не равно `123`, поэтому мы приводим все значения к строке, чтобы избежать багов типов (TypeScript/JS Strict Typing).
3. Используйте метод `Array.includes(element)` внутри оператора ветвления.
4. В блоке `if`: Инстанцируйте `ApiRange("A" + i)` и покрасьте через `SetFillColor()`.
5. В блоке `else` (Если не найдено): Добавьте элемент в кэш через `seen.push(docNum)`.

## 🧠 Глубинный разбор (Big-O и структуры данных)

Метод `seen.includes(ЭЛЕМЕНТ)` обходит плоский массив `O(N)`. Это означает, что для 10 000-й строки, парсер совершит 9 999 проверок. Суммарная сложность — `O(N^2)`.

Для 5-10 тысяч строк V8-движок обработает это мгновенно (в JavaScript массивы невероятно оптимизированы в С++ ядре браузера). Но если у вас **миллион** строк (Уровень Big Data), правильная Enterprise-практика предписывает применять структуру `Set` (Хэш-карту с `O(1)` доступом):

```javascript
// O(N) Архитектура.
let seen = new Set();
// ...
if (seen.has(docNum)) { 
    // Маркировка дубликата
} else {
    seen.add(docNum);
}
```

*Пока же мы применяем классический 1D-массив для закрепления основ синтаксиса `push` / `includes`.*

## ✍️ Эталонное решение (Синтаксис)

Для тех, кому нужна сверка синтаксиса:

```javascript
/* Строгое приведение к строке - Защита от багов */
let docNum = String(sheet.GetRange("A" + i).GetValue());

if (seen.includes(docNum)) {
    // Дубликат в кэше
    let targetRange = sheet.GetRange("A" + i);
    targetRange.SetFillColor(yellow);
    targetRange.SetBold(true); // UI Markup
} else {
    // Уникальный элемент - Сохраняем (Push to Memory)
    seen.push(docNum);
}
```

**Вопрос Архитектору:** Почему мы не используем `Delete("up")` прямо тут, внутри этого O(N) прохода?
Вспомните предыдущий урок! Это прямой Forward-цикл. Физическое удаление "на лету" сдвинет таблицу и пропустит следующую за дубликатом строку. Для физического удаления дубликатов используют два прохода: поиск индексов (`push` в мусорный накопитель), а затем `Reverse-loop` по сохраненным индексам для `Delete("up")`.
