# Урок 13. Группировки и обогащение таблиц

Вступление

Представьте аналитика в супермаркете, которому нужно понять структуру продаж не по отдельным чекам, а по категориям товаров, времени суток и типам покупателей одновременно. Простое суммирование здесь не поможет — нужно сначала сгруппировать данные по нескольким измерениям, вычислить промежуточные показатели для каждой группы, а затем проанализировать результат. В DAX функции SUMMARIZE, GROUPBY и ADDCOLUMNS выполняют именно эту работу — создают виртуальные сводные таблицы прямо внутри формул.

Табличные функции группировки открывают новое измерение в DAX — возможность создавать промежуточные наборы данных для сложных вычислений. Это мост между простыми агрегатами и продвинутой аналитикой, позволяющий решать задачи, невозможные с базовыми функциями. От расчета средневзвешенных показателей до создания динамических сегментов — все строится на виртуальных таблицах.

В предыдущих уроках мы научились фильтровать и модифицировать контекст. Теперь освоим создание собственных табличных структур для многоуровневых вычислений.

Теоретическая часть

Основные понятия

```dax
SUMMARIZE — это функция создания сводной таблицы внутри DAX, группирующая данные по указанным столбцам и опционально добавляющая агрегированные выражения. SUMMARIZE возвращает таблицу с уникальными комбинациями значений группирующих столбцов. Например, SUMMARIZE(Sales, Product[Category], Customer[Country]) создаст таблицу всех уникальных комбинаций категорий и стран из видимых данных. Это основа для создания промежуточных наборов данных в сложных вычислениях.
```

GROUPBY — альтернативная функция группировки, работающая исключительно с таблицами и не имеющая доступа к модели данных. GROUPBY требует, чтобы все используемые столбцы находились в одной таблице или были предварительно объединены. В отличие от SUMMARIZE, GROUPBY не может напрямую обращаться к связанным таблицам. Это делает GROUPBY более предсказуемой, но менее гибкой.

ADDCOLUMNS обогащает существующую таблицу новыми вычисляемыми столбцами. Функция принимает таблицу и набор пар “имя столбца - выражение”, добавляя результаты вычислений к каждой строке. ADDCOLUMNS сохраняет строковый контекст исходной таблицы, позволяя использовать значения существующих столбцов в вычислениях. Это ключевой инструмент для создания промежуточных вычислений перед финальной агрегацией.

Виртуальная таблица — это временная табличная структура, существующая только во время вычисления формулы. Виртуальные таблицы не сохраняются в модели данных, не занимают память постоянно, создаются и уничтожаются при каждом вычислении. Они служат промежуточным этапом для сложных вычислений, позволяя разбить задачу на логические шаги.

Ключевая механика

Механизм работы SUMMARIZE можно представить как автоматическое создание сводной таблицы Excel внутри формулы. Функция проходит по исходной таблице, находит все уникальные комбинации значений в указанных столбцах группировки, создает строку для каждой уникальной комбинации. Если указаны выражения для вычисления, они выполняются в контексте каждой группы. Важная особенность: SUMMARIZE может использовать столбцы из связанных таблиц благодаря автоматическому расширению через связи.

Представьте SUMMARIZE как сортировщика писем на почте. Он берет огромную кучу писем (исходные данные), раскладывает их по ячейкам согласно адресам (группировка), подсчитывает количество в каждой ячейке или выполняет другие операции (агрегация). В отличие от ручной сортировки, SUMMARIZE делает это мгновенно и может создавать многоуровневые группировки — сначала по городу, потом по улице, потом по дому.

GROUPBY работает иначе — как строгий бухгалтер, который работает только с одной книгой учета. Он не может заглядывать в другие книги (связанные таблицы), но зато его расчеты всегда предсказуемы и не зависят от внешних факторов. GROUPBY требует явного указания агрегатной функции через CURRENTGROUP() для доступа к строкам текущей группы.

Контекстные особенности табличных функций часто вызывают путаницу. SUMMARIZE может “ломать” ожидаемый контекст при использовании в CALCULATE, создавая неожиданные результаты. Это происходит из-за того, что SUMMARIZE создает собственный контекст оценки для выражений. ADDCOLUMNS более предсказуема — она сохраняет строковый контекст для каждой строки таблицы.

Композиция табличных функций позволяет создавать сложные конструкции. Можно использовать FILTER над SUMMARIZE, передать результат в ADDCOLUMNS, а затем агрегировать итератором. Это открывает безграничные возможности для аналитики, но требует понимания потока данных между функциями.

Синтаксис и правила

Базовый синтаксис SUMMARIZE: SUMMARIZE(table, groupBy_column1, groupBy_column2, …, name1, expression1, …). Первый параметр — исходная таблица. Далее идут столбцы для группировки из этой таблицы или связанных таблиц. Опционально можно добавить пары “имя-выражение” для вычисляемых столбцов. Пример: SUMMARIZE(Sales, Product[Category], “Total”, SUM(Sales[Amount])).

GROUPBY имеет строгий синтаксис: GROUPBY(table, groupBy_column1, …, name1, CURRENTGROUP(), expression1). Все столбцы группировки должны быть из указанной таблицы. Для агрегации обязательно использование CURRENTGROUP(), который представляет строки текущей группы. Пример: GROUPBY(Sales, Sales[ProductID], “Count”, COUNTX(CURRENTGROUP(), Sales[OrderID])).

```dax
ADDCOLUMNS принимает таблицу и пары имя-выражение: ADDCOLUMNS(table, name1, expression1, name2, expression2, …). Выражения вычисляются в строковом контексте каждой строки таблицы. Можно ссылаться на существующие столбцы таблицы. Пример: ADDCOLUMNS(Products, “Margin”, Products[ListPrice] - Products[Cost]).
```

Важное правило: табличные функции возвращают таблицы, поэтому их нельзя использовать напрямую как меры. Нужен агрегатор (SUM, COUNT, итератор) или CALCULATE для преобразования таблицы в скалярное значение. Например: COUNTROWS(SUMMARIZE(Sales, Product[Category])) подсчитает количество уникальных категорий.

Сравнение с альтернативами

В Excel создание сводных таблиц происходит через интерфейс, результат статичен и требует обновления при изменении данных. DAX-функции создают виртуальные сводные динамически при каждом вычислении. Одна формула DAX может заменить десятки промежуточных сводных таблиц Excel.

SQL GROUP BY концептуально похож на SUMMARIZE/GROUPBY, но выполняется на стороне базы данных при запросе. DAX работает с данными в памяти, обеспечивая мгновенный отклик. SQL требует явного указания всех агрегатных функций, DAX может создавать группировку без агрегации.

Pandas groupby() в Python предлагает похожую функциональность с методами agg(), transform(). Но pandas требует явного кода для каждого шага, DAX позволяет вложенные конструкции в одной формуле. DAX оптимизирован для интерактивной аналитики, pandas — для программной обработки.

Частые ошибки

Использование SUMMARIZE для простой группировки без агрегации часто приводит к избыточности. Если нужны только уникальные значения, используйте VALUES или DISTINCT — они эффективнее. SUMMARIZE оправдан при необходимости группировки по нескольким столбцам или добавлении вычислений.

Неправильное понимание контекста в SUMMARIZE создает неожиданные результаты. Выражения в SUMMARIZE вычисляются в контексте группы, а не всей таблицы. Это может привести к некорректным расчетам при использовании мер. Для надежности используйте ADDCOLUMNS после SUMMARIZE.

Забывание о необходимости агрегации табличного результата — синтаксическая ошибка новичков. Попытка использовать SUMMARIZE(…) как меру вызовет ошибку. Всегда оборачивайте табличные функции в агрегаторы или итераторы для получения скалярного значения.

Избыточная вложенность табличных функций усложняет код и замедляет вычисления. Часто можно упростить конструкцию, переосмыслив логику. Если формула занимает больше 5-6 строк с множественными вложениями, стоит разбить на промежуточные меры.
