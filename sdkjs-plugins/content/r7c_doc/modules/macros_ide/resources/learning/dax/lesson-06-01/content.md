# Урок 21. Базовые функции Time Intelligence

Вступление

Представьте ритейлера, анализирующего праздничные продажи. Ему нужно сравнить декабрьские продажи этого года с прошлым, учесть, что Черная пятница сдвинулась на неделю, и понять реальный рост без сезонных искажений. Простое сравнение чисел даст неверную картину — нужны интеллектуальные временные функции, понимающие структуру календаря и бизнес-логику периодов. DAX предлагает элегантные решения через функции Time Intelligence, автоматически выравнивающие и сравнивающие временные периоды.

Временной анализ — это основа управленческой отчетности. Сравнение с прошлым годом, квартальная динамика, сезонные тренды — все это требует специализированных функций. SAMEPERIODLASTYEAR, DATEADD и PARALLELPERIOD позволяют легко манипулировать временными периодами, создавая сложные сравнения одной строкой кода. Без этих функций пришлось бы писать громоздкие формулы с множественными условиями.

В предыдущих уроках мы создали календарь и изучили полусуммируемые показатели. Теперь используем эти знания для полноценного временного анализа.

Теоретическая часть

Основные понятия

SAMEPERIODLASTYEAR — функция, возвращающая даты за аналогичный период прошлого года. Если текущий контекст содержит январь 2014, функция вернет январь 2013. Для квартала вернет тот же квартал прошлого года. SAMEPERIODLASTYEAR автоматически учитывает високосные годы и разное количество дней в месяцах, обеспечивая корректное сравнение периодов.

```dax
DATEADD — универсальная функция сдвига дат на заданный интервал. DATEADD может сдвигать даты на дни, месяцы, кварталы или годы вперед или назад. DATEADD(‘Calendar’[Date], -1, MONTH) вернет даты предыдущего месяца. Функция интеллектуально обрабатывает границы периодов: сдвиг 31 января на месяц вперед даст 28/29 февраля, а не несуществующую дату.
```

PARALLELPERIOD — функция для получения полного параллельного периода. В отличие от DATEADD, которая сдвигает каждую дату индивидуально, PARALLELPERIOD возвращает весь период целиком. PARALLELPERIOD(‘Calendar’[Date], -1, QUARTER) вернет полный предыдущий квартал, даже если текущий контекст содержит только часть квартала.

Временной интеллект (Time Intelligence) — это набор функций DAX, специально разработанных для работы с временными данными. Эти функции понимают структуру календаря, иерархию периодов (год-квартал-месяц-день) и автоматически обрабатывают особенности календаря. Time Intelligence требует правильно настроенной таблицы дат с непрерывной последовательностью.

Ключевая механика

Механизм работы временных функций основан на манипулировании набором дат в фильтровом контексте. Функция получает текущий набор дат, применяет трансформацию (сдвиг, выбор периода) и возвращает новый набор дат. Этот новый набор используется как фильтр для вычисления метрик. Важно: функции возвращают таблицу дат, а не скалярное значение.

Представьте временные функции как машину времени для данных. SAMEPERIODLASTYEAR переносит вас ровно на год назад в тот же день. DATEADD — это регулируемая машина времени, где вы выбираете направление и дистанцию. PARALLELPERIOD — это телепорт в другой период целиком, независимо от вашего текущего положения внутри периода.

Обработка границ периодов — критический аспект временных функций. При сдвиге дат функции интеллектуально обрабатывают несуществующие даты. Например, при сдвиге 31 марта на месяц назад получится 28/29 февраля, а не ошибка. Это обеспечивает робастность расчетов при любых входных данных.

Зависимость от календаря делает настройку таблицы дат критически важной. Временные функции работают только с правильно настроенным календарем: непрерывная последовательность дат, корректные связи с фактами, отметка таблицы как календарной. Пропуски в календаре или неправильные типы данных приведут к некорректным результатам или ошибкам.

Контекстная адаптация позволяет одной формуле работать на разных уровнях детализации. Мера с SAMEPERIODLASTYEAR автоматически адаптируется: показывает прошлый год при годовом фильтре, прошлый месяц того же названия при месячном фильтре, вчерашний день год назад при дневном фильтре.

Синтаксис и правила

```dax
SAMEPERIODLASTYEAR имеет простой синтаксис с одним параметром: SAMEPERIODLASTYEAR(dates_column). Параметр должен быть столбцом дат из таблицы календаря. Функция возвращает таблицу дат, обычно используется внутри CALCULATE:
[Sales Last Year] :=
CALCULATE(
SUM(FactInternetSales[SalesAmount]),
SAMEPERIODLASTYEAR('DimDate'[FullDateAlternateKey])
)
DATEADD требует три параметра: DATEADD(dates_column, number_of_intervals, interval). Интервал может быть DAY, MONTH, QUARTER, YEAR. Число может быть положительным (вперед) или отрицательным (назад):
[Sales Previous Month] :=
CALCULATE(
SUM(FactInternetSales[SalesAmount]),
DATEADD('DimDate'[FullDateAlternateKey], -1, MONTH)
)
```

PARALLELPERIOD также имеет три параметра: PARALLELPERIOD(dates_column, number_of_intervals, interval). Синтаксис похож на DATEADD, но семантика отличается — возвращается полный период:

```dax
[Sales Previous Quarter Full] :=
CALCULATE(
SUM(FactInternetSales[SalesAmount]),
PARALLELPERIOD('DimDate'[FullDateAlternateKey], -1, QUARTER)
)
```

Сравнение с альтернативами

В Excel для сравнения периодов используются функции СУММЕСЛИМН с множественными условиями на даты или сложные формулы с СМЕЩ. Это требует ручного расчета границ периодов и не адаптируется к изменению контекста. DAX автоматизирует процесс и динамически адаптируется к фильтрам.

SQL подходы включают самосоединения таблиц с условиями на даты или оконные функции LAG/LEAD. Но SQL-запросы статичны и требуют переписывания для разных периодов. DAX функции работают универсально для любого уровня временной детализации.

Power Query может подготовить данные с предрассчитанными сравнениями периодов, но это увеличивает объем данных и снижает гибкость. Любое изменение в логике сравнения требует перезагрузки данных. DAX вычисляет сравнения на лету без дублирования данных.

Частые ошибки

Использование временных функций без правильного календаря — главная причина ошибок. Функции требуют непрерывную последовательность дат. Пропуски приведут к некорректным результатам. Всегда проверяйте целостность календаря перед использованием Time Intelligence.

Путаница между DATEADD и PARALLELPERIOD создает неверные результаты. DATEADD сдвигает текущий контекст, PARALLELPERIOD возвращает полный период. Для частичного месяца DATEADD вернет частичный сдвинутый месяц, PARALLELPERIOD — полный предыдущий месяц.

Неправильный выбор интервала в DATEADD может дать неожиданные результаты. Сдвиг на месяц не равен сдвигу на 30 дней. Месяцы имеют разную длину, и DATEADD учитывает это. Выбирайте интервал согласно бизнес-логике, а не календарным дням.

Игнорирование отсутствующих данных в прошлых периодах приводит к некорректным сравнениям. Если данные начинаются с середины 2011 года, сравнение января 2011 с январем 2010 даст BLANK. Добавляйте проверки на наличие данных в сравниваемом периоде.

## DAX-решение

```dax
// Продажи текущего периода
[Sales Current] :=
SUM(FactInternetSales[SalesAmount])
// Продажи за аналогичный период прошлого года
[Sales LY] :=
CALCULATE(
[Sales Current],
SAMEPERIODLASTYEAR('DimDate'[FullDateAlternateKey])  // сдвиг на год назад
)
// Изменение год к году
[YoY Change %] :=
DIVIDE(
[Sales Current] - [Sales LY],
[Sales LY],
0
)
```

Проверка результата: 2013: Current = $9,791,060.30, LY = $6,530,344.23, YoY = +50.0% 2012: Current = $6,530,344.23, LY = $12,093,738.44, YoY = -46.0% Январь 2013: Current = $673,124.93, LY = $601,085.91, YoY = +12.0% Категория Bikes 2013: Current = $9,486,365.45, LY = $6,298,558.84 Функция корректно находит параллельные периоды прошлого года

Ответы

```dax
SAMEPERIODLASTYEAR вернет даты февраля 2013 года (аналогичный месяц прошлого года).
```

DATEADD сдвигает текущий контекст, сохраняя его границы; PARALLELPERIOD возвращает полный параллельный период.

```dax
DATEADD принимает столбец дат, число интервалов (может быть отрицательным) и тип интервала (DAY/MONTH/QUARTER/YEAR).
DATEADD вернет 28 или 29 февраля (последний день февраля), интеллектуально обрабатывая границы.
```

Непрерывная последовательность дат, правильные типы данных, связь с фактами, отметка как Date Table.

Да, DATEADD может сдвигать в обоих направлениях: положительное число — вперед, отрицательное — назад.

PARALLELPERIOD вернет полный предыдущий месяц, независимо от того, частичный или полный текущий контекст.
