# Урок 22. Инженерия данных: Агрегация (Сводка) множества листов с помощью массивов

Вы дошли до середины Модуля 4. Это означает, что вы уже понимаете навигацию через `Api.GetSheets()`. 

В этом теоретическом уроке мы спроектируем **Макрос-Сборщик (Aggregator)**. Скрипт, который бежит по 50 одинаковым листам (филиалам), извлекает данные и формирует консолидированный "Свод". Как разработчик VBA вы, вероятно, писали это не раз, используя `Cells(i, 1).Value` или копируя диапазоны через Буфер Обмена.
Здесь мы покажем архитектуру массовой сборки на JavaScript в среде Р7-Офис.

---

## 🚀 Архитектура 1: Сбор "Итогов" (Сканирование вкладок)

Разберем скрипт-сканер, который пробегается по всем листам книги, забирает конкретную ячейку (например, Итоговую Выручку в `B2`) и агрегирует эти цифры на Сводном листе, сдвигая курсор (`writeRow`). Обратите внимание на исключение системного листа `Сводка` из цикла:

```javascript
(function() {
    let allSheets = Api.GetSheets();
    
    // Пул Сводного листа + Фабрика, если его нет
    let summarySheet = Api.GetSheet("Сводка");
    if (summarySheet === null) {
        summarySheet = Api.AddSheet("Сводка");
    } else {
        summarySheet.GetRange("A1:B1000").Clear(); // Очистка старых сводов
    }
    
    summarySheet.GetRange("A1").SetValue("Филиал");
    summarySheet.GetRange("B1").SetValue("Выручка");
    
    let writeRow = 2; // Курсор (Указатель строки)
    
    // Aggregation Loop (Scan)
    for (let i = 0; i < allSheets.length; i++) {
        let currentSheet = allSheets[i];
        let sName = currentSheet.GetName();
        
        // Фильтр системных листов
        if (sName !== "Сводка" && sName !== "Настройки") {
            
            let revenue = currentSheet.GetRange("B2").GetValue();
            
            // Вставка по одной ячейке (Только для сканеров итогов!)
            summarySheet.GetRange("A" + writeRow).SetValue(sName);
            summarySheet.GetRange("B" + writeRow).SetValue(revenue);
            
            writeRow++; // Инкремент каретки
        }
    }
})();
```

Но что, если филал присылает не одно число (Итог), а целую реестровую таблицу продаж на 5000 строк? 
Обращение к `SetValue` в цикле для каждой из 50 000 строк в Р7-Офис (как и в Excel) приведет к жестокому зависанию (Performance Bottleneck).

Вам нужен **Двумерный массив (Matrix ETL)**.

---

## ⚡ Оптимизация 2D Array: Batch Read/Write (Массовый GetValue)

Метод `GetRange("A1:C5000").GetValue()` невероятно умен. Когда вы передаете ему не отдельную ячейку, а гигантский диапазон, он возвращает слепок данных в виде классического Двумерного Массива JavaScript `(Массив Массивов)` — `[ ["A1", "B1"], ["A2", "B2"] ]`.

В VBA вы писали:
`Dim arr As Variant`
`arr = Range("A1:C5000").Value`

В Р7 это выглядит так же гармонично.
Как только данные "всосались" в ОЗУ:
1. Вы производите хирургическую фильтрацию мусора прямо в памяти операционными циклами (Это происходит за миллисекунды, так как нет обращений к графику DOM).
2. Вычисляете целевой размер диапазона на "Сводке".
3. Производите залп обратно в ячейки с помощью `SetValue(filteredArray)`.

### Пример: Трансформация Матрицы (Cleanse Data)

Давайте напишем скрипт-чистильщик. Он читает монолит (500 строк), отбрасывает пустые строки в памяти и выгружает плотные данные. Идентично процедуре "Удалить пустые строки", но работает моментально за счет RAM:

```javascript
(function() {
    let sheet = Api.GetActiveSheet();
    
    // Забираем грязную таблицу (Batch Read)
    let data = sheet.GetRange("A1:C500").GetValue(); 
    let cleanData = []; // Резервуар для валидных записей
    
    // Перебор строк массива data. i = Строка. [0] = Колонка A.
    for (let i = 0; i < data.length; i++) {
        let row = data[i]; 
        let valA = row[0]; // Ключевой столбец А
        
        // Если valA не undefined/null/empty
        if (valA) {
            cleanData.push(row);
        }
    }
    
    // Расчет целевого адреса сброса (Важно совпадение ширины и высоты рамки)
    let targetLoc = sheet.GetRange("F1:H" + cleanData.length); 
    
    // Batch Write - Залп!
    targetLoc.SetValue(cleanData);
})();
```

Разберите этот паттерн (ETL: Extract, Transform, Load). Без понимания Batching-архитектуры профессиональная разработка в Р7-Офис невозможна.

## 💥 Различия между `.Copy()` и `.SetValue(2D_Matrix)`

В Уроке 12 мы изучали `.Copy()`. Почему при сборе агрегатов мы используем `GetValue/SetValue`?

1. Метод **Copy**:
   * Тянет за собой форматы (Цвета, Borders) и сохраняет **Рабочие формулы**.
   * Идеален для визуального клонирования.
   * Вы не можете мутировать или фильтровать данные в процессе полета — это "глухой ксерокс".

2. Метод **Матричного GetValue / SetValue**:
   * Вытаскивает только **"голые значения"** (Hardcoded values). 
   * Формулы (SUM, VLOOKUP) вычисляются, и в память переходит лишь их цифровой ответ. Форматирование теряется.
   * Идеален для СУБД, агрегации и математических фильтров в ОЗУ. Вы можете менять ячейки, умножать курсы валют на лету (`row[1] *= 100`) и выбрасывать мусор до того, как информация ляжет на чистый лист.
