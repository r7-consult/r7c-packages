## 🏛️ Механика Эталона (Разбор структуры)

Откройте любой официальный шаблон от разработчика (например, макрос `AddSheet`). Он выглядит огромным, но на самом деле полезного кода там всего **одна-две строчки**! Всё остальное — это броня инженера (Защита от сбоев). 

Давайте разберем его по частям.

### 1. Купол (IIFE)
Почти любой шаблонный скрипт начинается и заканчивается так:
```javascript
(function() {
    // Весь код
})();
```
Это IIFE (Немедленно Вызываемая Функция). Она нужна для изоляции памяти. Переменная `let sheet`, объявленная внутри, умрет сразу после завершения скрипта и не сломает другие макросы в документе. Всегда копируйте эту оболочку!

### 2. Строгий Режим (Strict Mode)
В самом верху внутри функции вы найдете:
```javascript
'use strict';
```
Система безопасности JavaScript. Если вы напишете `i = 5` не написав заранее `let i` или `var i`, в обычном режиме браузер простит вас и создаст глобальную уязвимую переменную. Режим `'use strict'` заблокирует скрипт с криком "Вы забыли объявить переменную!". Это защита от классических опечаток.

### 3. Блок-предохранитель (Try / Catch)
Архитекторы знают, что макрос может упасть (например, вы пытаетесь выделить ячейку, а пользователь вообще не открыл таблицу, или API редактора "подвис"). На серверном уровне (Backend) падения нельзя допускать без перехвата.

```javascript
try {
    // Попытайся выполнить этот опасный код
    const api = Api; 
    let sheet = Api.GetActiveSheet();
    // ... логика ...
    
    console.log('Macro executed successfully'); // Если дошли сюда - все ОК
    
} catch (error) {
    // Если в блоке TRY произошла ошибка, скрипт НЕ ломает программу!
    // Он падает СЮДА (в сетку), и переменная error содержит текст ошибки.
    
    console.error('Macro execution failed: ', error.message);
    
    // Продвинутый трюк шаблона: Вывели текст ошибки прямо в ячейку A1!
    let sheet = Api.GetActiveSheet();
    if (sheet) {
        sheet.GetRange("A1").SetValue("Error: " + error.message);
    }
}
```

Именно эта "броня" из `Try/Catch` и распухает шаблон на десятки строк! Но если выбросить броню, мы увидим Ядро.

## 🔍 Напутствие разработчику: Умейте искать Ядро

Когда вы скачиваете шаблон **AddComment_macroR7.js**, не пугайтесь его размера. 
Отбросьте `(function() {`, отбросьте `'use strict';`, отбросьте комментарии автора `@author R7-Consult` и блок `catch (error)`.

Посмотрите внутрь `try`:
```javascript
// Add two comments to the document
Api.AddComment("Comment 1", "Bob");
Api.AddComment("Comment 2");

let comments = Api.GetComments();
let worksheet = Api.GetActiveSheet();

worksheet.GetRange("A1").SetValue("Text: " + comments[0].GetText());
```

**Вот они! Ценнейшие методы API, которые мы искали!**
Оказывается, чтобы добавить комментарий, не нужно брать никакие диапазоны, метод вызывается глобально `Api.AddComment()`. И оказывается, комментарии собираются в Массив (объект Collection) через `Api.GetComments()`.

**Главный урок этого модуля:** Шаблонные макросы — это ваш справочник. Если вам нужно понять, как запустить команду (Добавить лист, Найти пересечения диапазонов Intersect, Пересчитать расчеты RecalculateAllFormulas) — просто откройте пример, найдите Ядро внутри `try/catch` и скопируйте эту логику себе.
