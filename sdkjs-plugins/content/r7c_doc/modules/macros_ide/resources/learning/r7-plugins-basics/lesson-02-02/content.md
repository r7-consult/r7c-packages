**Урок 2.2. Создание конфигурационного файла config.json (включая генерацию GUID)**

Файл **config.json** — это «паспорт» плагина. Он должен быть валидным JSON-документом и содержать следующие ключевые поля:

json

**{**

**"name": "Плагин окрашивания ячейки",**

**"guid": "asc.{6401CE6B-3E19-45E1-9352-BFCF41989AA5}",**

**"version": "0.0.1",**

**"variations": \[**

**{**

**"description": "Учебный плагин",**

**"url": "index.html",**

**"icons": \["logo(64x64).png"\],**

**"isViewer": true,**

**"EditorsSupport": \["cell"\],**

**"isVisual": true,**

**"isModal": false,**

**"isInsideMode": true,**

**"buttons": \[\]**

**}**

**\]**

**}**

**Обязательные требования:**

-    Поле **guid** должно содержать **глобально уникальный идентификатор** в формате asc.{...}.  
    Пример: asc.{6401CE6B-3E19-45E1-9352-BFCF41989AA5}.
-    Для генерации GUID можно использовать онлайн-сервисы (например, [https://www.guidgenerator.com/](https://www.guidgenerator.com/)).
-    Значение **url** должно точно совпадать с именем HTML-файла (обычно index.html).
-    Поле **EditorsSupport** указывает, в каких редакторах работает плагин: \["cell"\] — для табличного редактора.

Неверный формат GUID или отсутствие обязательных полей приведёт к ошибке при установке.
