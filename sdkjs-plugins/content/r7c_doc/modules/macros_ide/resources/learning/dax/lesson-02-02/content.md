# Урок 5. Меры против вычисляемых столбцов

Вступление

Представьте два ресторана: в первом повар готовит каждое блюдо по индивидуальному заказу, во втором все блюда заранее приготовлены и ждут в витрине. Первый подход гибкий — блюдо всегда свежее и может быть адаптировано под вкус клиента, но требует времени на приготовление. Второй быстрый — взял и пошел, но блюдо может оказаться не таким свежим и изменить его невозможно. Точно такая же дилемма стоит перед каждым DAX-разработчиком: вычислять значения динамически через меры или сохранить их заранее в вычисляемых столбцах.

Неправильный выбор между мерой и столбцом может привести к катастрофическим последствиям. Модель данных разбухает до гигабайтов вместо мегабайтов, отчеты обновляются минутами вместо секунд, а изменение бизнес-логики требует полной перестройки модели. Понимание фундаментальной разницы между этими подходами определяет успех всего аналитического решения.

В предыдущих уроках мы изучили основы DAX, синтаксис и типы данных. Теперь пришло время применить эти знания для принятия архитектурных решений о структуре вычислений.

Теоретическая часть

Основные понятия

```dax
Мера (Measure) — это формула DAX, которая вычисляется динамически в момент использования в отчете. Мера не хранит данные, она хранит только инструкцию для вычисления. Представьте калькулятор, который всегда готов выполнить операцию, но не запоминает результат. Каждый раз, когда вы меняете фильтры в сводной таблице, мера пересчитывается заново. Например, мера [Total Sales] := SUM(FactInternetSales[SalesAmount]) будет показывать разные значения в зависимости от выбранного года, категории продукта или региона.
```

Вычисляемый столбец (Calculated Column) — это новый столбец в таблице, где значение вычисляется для каждой строки при обновлении модели данных. После вычисления значения сохраняются в модели как обычные данные. Это как добавить новую колонку в Excel-таблицу с формулой, которая вычисляется один раз и сохраняет результат. Например, столбец Profit = FactInternetSales[SalesAmount] - FactInternetSales[TotalProductCost] создаст новый столбец с прибылью для каждой транзакции.

Контекст вычисления определяет, какие данные участвуют в расчете. Для мер это фильтры отчета — выбранные годы, продукты, клиенты. Для столбцов это текущая строка таблицы. Мера видит отфильтрованные данные, столбец видит только значения своей строки. Эта разница фундаментальна и определяет возможности использования каждого подхода.

Хранение и производительность — критический фактор выбора. Меры не занимают места в модели, но требуют вычислительных ресурсов при каждом использовании. Столбцы занимают память пропорционально количеству строк и уникальности значений, но возвращают результат мгновенно. В таблице с миллионом строк дополнительный столбец может увеличить модель на десятки мегабайт.

Ключевая механика

Механизм работы мер основан на динамическом вычислении в контексте фильтрации. Когда вы помещаете меру в сводную таблицу, DAX автоматически применяет все активные фильтры — из срезов, из строк и столбцов сводной таблицы, из визуальных фильтров. Мера получает отфильтрованную таблицу и выполняет вычисление только для видимых данных. Это происходит каждый раз при изменении любого фильтра.

Представьте меру как умного помощника, который каждый раз пересчитывает результат, учитывая текущую ситуацию. Вы спрашиваете “какая сумма продаж?”, и помощник уточняет: “За какой период? По каким продуктам? Для каких клиентов?” — затем идет к данным, применяет все условия и возвращает точный ответ. Это занимает время, но результат всегда актуален и точен для текущего контекста.

Вычисляемые столбцы работают принципиально иначе. При создании столбца DAX проходит по каждой строке таблицы и вычисляет значение, используя только данные этой строки и связанных таблиц. Результат сохраняется и становится частью модели. Это как заранее подготовленные ответы на все возможные вопросы — быстро, но негибко.

Критическое различие проявляется при агрегировании. Мера может использовать любую агрегатную функцию и автоматически учтет контекст. Столбец содержит детальные значения, которые потом можно агрегировать стандартными способами (сумма, среднее, количество). Но столбец не может динамически менять логику вычисления в зависимости от контекста — что вычислено при создании, то и хранится.

Обновление данных также работает по-разному. Меры не требуют пересчета при обновлении модели — они всегда вычисляются на актуальных данных. Столбцы пересчитываются полностью при каждом обновлении модели, что может занимать значительное время для больших таблиц. Если исходные данные меняются часто, множество вычисляемых столбцов может существенно замедлить обновление.

Синтаксис и правила

```dax
Синтаксис создания меры использует оператор := и обычно начинается с квадратных скобок вокруг имени. Меры создаются в области вычислений Power Pivot или в строке формул. Правильный синтаксис: [Measure Name] := EXPRESSION. Мера может быть создана в любой таблице или в специальной таблице мер, но это влияет только на организацию, не на функциональность.
```

Вычисляемые столбцы создаются прямо в таблице через интерфейс Power Pivot. В представлении данных выберите пустой столбец справа от существующих данных и введите формулу в строку формул. Синтаксис: ColumnName = EXPRESSION. Обратите внимание — для столбцов используется одинарный знак равенства, не двоеточие-равно как для мер.

При ссылке на другие столбцы в формуле вычисляемого столбца можно опускать имя таблицы, если столбец находится в той же таблице. Но для ясности рекомендуется всегда указывать полную ссылку: TableName[ColumnName]. В мерах всегда необходимо указывать полное имя таблицы и столбца.

Важное правило: меры могут ссылаться на другие меры, но вычисляемые столбцы не могут ссылаться на меры. Это логично — столбец вычисляется один раз при обновлении модели, когда нет контекста фильтрации для вычисления меры. Меры же могут использовать вычисляемые столбцы как обычные столбцы данных.

Сравнение с альтернативами

В классическом Excel формула в ячейке может быть как статической (вычисляется один раз), так и динамической (пересчитывается при изменении исходных данных). В DAX это разделение более строгое: столбцы всегда статические относительно фильтров, меры всегда динамические. Это делает поведение более предсказуемым, но требует осознанного выбора.

Power Query (редактор запросов) также может создавать вычисляемые столбцы при загрузке данных. Разница в том, что столбцы Power Query вычисляются до загрузки в модель и не могут использовать DAX-функции или ссылаться на другие таблицы модели. Вычисляемые столбцы DAX создаются после загрузки и могут использовать связи модели.

SQL-запросы традиционно разделяют вычисления на уровне строк (в SELECT) и агрегированные вычисления (с GROUP BY). DAX-столбцы соответствуют вычислениям уровня строк, меры — агрегированным вычислениям. Но в отличие от SQL, где оба типа вычислений происходят при выполнении запроса, в DAX столбцы материализуются в модели.

Частые ошибки

Создание вычисляемого столбца для значений, которые нужно агрегировать — самая распространенная ошибка. Новички часто создают столбец TotalSales в таблице фактов, умножая количество на цену. Это работает, но неэффективно — столбец займет много места, а для анализа все равно придется его суммировать. Правильнее создать меру, которая выполнит умножение и суммирование одновременно.

Попытка использовать меру в вычисляемом столбце приводит к ошибке или неожиданным результатам. Поскольку столбец вычисляется без контекста фильтрации, мера вернет значение для всей таблицы, а не для конкретной строки. Это частая причина путаницы, когда во всех строках столбца появляется одинаковое значение.

Избыточное использование столбцов для промежуточных вычислений раздувает модель. Если столбец нужен только как промежуточный шаг для меры, лучше выполнить вычисление прямо в мере. Каждый дополнительный столбец увеличивает размер модели и время обновления.

Игнорирование влияния на производительность при выборе между мерой и столбцом может сделать модель неработоспособной. Сложная мера, вычисляющаяся секундами, сделает отчет неинтерактивным. Десятки вычисляемых столбцов могут увеличить модель до размера, превышающего доступную память.

Ответы

Мера вычисляется динамически при каждом использовании в отчете, столбец — один раз при обновлении модели данных.

```dax
Для создания меры используется оператор := (двоеточие-равно), для столбца — простой знак = (равно).
```

Нельзя эффективно использовать меру в столбце, так как отсутствует контекст фильтрации при вычислении столбца.

Для фильтров в срезах необходим вычисляемый столбец, так как меры не могут использоваться для фильтрации.

Вычисляемый столбец увеличивает размер модели пропорционально количеству уникальных значений и строк в таблице.

Да, мера может использовать вычисляемый столбец как обычный столбец данных в любых агрегатных функциях.

Изменение формулы меры не требует пересчета модели, изменение применяется мгновенно во всех отчетах.
