# Урок 6. Базовые агрегаты

Вступление

Представьте склад огромной корпорации с миллионами товаров. Директору не нужно знать каждую коробку — ему важны итоги: общее количество, средняя стоимость, минимальный остаток. Без агрегации данных он утонет в деталях и не сможет принять ни одного решения. Точно так же работает бизнес-аналитика: из океана транзакций нужно извлечь понятные метрики. Базовые агрегатные функции DAX — это фундамент любого анализа, превращающий сырые данные в управленческую информацию.

Эти функции настолько важны, что присутствуют в любой системе анализа данных — от Excel до промышленных баз данных. Но в DAX они работают умнее: автоматически учитывают контекст фильтрации, игнорируют пустые значения, адаптируются к структуре отчета. Освоив всего шесть функций, вы сможете ответить на 80% аналитических вопросов бизнеса.

В предыдущих уроках мы изучили основы языка DAX и разницу между мерами и столбцами. Теперь пришло время создать арсенал функций для решения реальных бизнес-задач.

## Что вы освоите в этом уроке

- Научитесь использовать функции SUM, AVERAGE для числовых агрегаций
- Освоите MIN и MAX для поиска экстремальных значений
- Изучите различия между COUNT, COUNTA, DISTINCTCOUNT и COUNTROWS
- Поймете паттерны безопасных вычислений и обработки пустых значений

В следующем уроке мы перейдем к итераторам семейства X, которые позволят выполнять построчные вычисления перед агрегацией.

Теоретическая часть

Основные понятия

Агрегатная функция — это операция, которая принимает множество значений и возвращает одно итоговое значение. В контексте DAX агрегатные функции работают со столбцами таблиц, автоматически учитывая текущий контекст фильтрации. Например, SUM(‘Sales’[Amount]) суммирует не весь столбец Amount, а только те строки, которые видны в текущем контексте — выбранном периоде, категории товаров, регионе. Это фундаментальное отличие от Excel, где СУММ работает с фиксированным диапазоном.

BLANK-значения — особый тип данных в DAX, представляющий отсутствие информации. Все базовые агрегатные функции автоматически игнорируют BLANK при вычислениях. Если в столбце из 100 значений 10 являются BLANK, функция SUM просуммирует только 90 непустых значений, AVERAGE вычислит среднее по 90 значениям, COUNT подсчитает 90 элементов. Это поведение защищает от искажения результатов неполными данными.

Скалярное значение — это единичный результат агрегатной функции: одно число, одна дата, один текст. Все базовые агрегаты возвращают скалярные значения, что позволяет использовать их в арифметических выражениях и как аргументы других функций. Например, результат SUM можно разделить на результат COUNT для получения среднего значения.

Гранулярность агрегации определяется контекстом использования функции. В общем итоге сводной таблицы агрегат работает со всеми данными. В ячейке на пересечении строки и столбца — только с данными, соответствующими обоим измерениям. Эта автоматическая адаптация делает одну меру универсальной для любого уровня детализации.

Ключевая механика

Механизм работы агрегатных функций в DAX основан на двухэтапном процессе. Сначала применяется контекст фильтрации — из всей таблицы выбираются только строки, удовлетворяющие текущим фильтрам. Затем к отфильтрованному набору применяется агрегатная операция. Этот процесс происходит автоматически и невидимо для пользователя, но понимание механики критично для написания эффективных формул.

Представьте агрегатную функцию как умного бухгалтера, который получает задание: “Посчитай сумму продаж”. Бухгалтер сначала уточняет контекст: “За какой период? По каким товарам? В каких регионах?” Получив ответы из фильтров отчета, он отбирает только нужные документы и выполняет подсчет. При изменении любого фильтра бухгалтер повторяет процесс с новыми критериями.

Различие между функциями подсчета часто вызывает путаницу. COUNT считает непустые значения в столбце чисел или дат. COUNTA считает непустые значения любого типа, включая текст. COUNTROWS считает строки в таблице независимо от содержимого. DISTINCTCOUNT считает уникальные непустые значения. Выбор правильной функции критичен для точности расчетов.

Обработка пустых и нулевых значений — важнейший аспект агрегации. BLANK и ноль — это разные значения в DAX. Ноль участвует в вычислениях как обычное число: учитывается в COUNT, влияет на AVERAGE, может быть минимумом в MIN. BLANK полностью игнорируется всеми агрегатными функциями. Это позволяет корректно обрабатывать неполные данные без дополнительных проверок.

Производительность агрегатных функций оптимизирована движком VertiPaq. Данные хранятся в сжатом колоночном формате, что делает операции суммирования и подсчета молниеносными даже на миллионах строк. Однако DISTINCTCOUNT требует больше ресурсов, так как должен отслеживать уникальность значений. При работе с большими объемами данных это может влиять на скорость отчетов.

Синтаксис и правила

Базовый синтаксис всех агрегатных функций однотипен: ФУНКЦИЯ(столбец). Столбец должен быть указан с именем таблицы: SUM(FactInternetSales[SalesAmount]). Это обязательное требование DAX, даже если имя столбца уникально во всей модели. Функции не принимают дополнительных параметров фильтрации — фильтрация происходит через контекст.

```dax
SUM и AVERAGE работают только с числовыми столбцами. При попытке применить их к тексту или логическим значениям возникнет ошибка. MIN и MAX работают с числами, датами и даже текстом (в алфавитном порядке). Это позволяет находить самую раннюю дату, самое большое число или последнее значение по алфавиту.
COUNT имеет особенность — работает только со столбцами чисел, дат и логических значений, игнорируя текст. Если нужно подсчитать текстовые значения, используйте COUNTA. Для подсчета всех строк независимо от содержимого применяйте COUNTROWS с таблицей в качестве аргумента: COUNTROWS(FactInternetSales).
DISTINCTCOUNT требует особого внимания при работе с ключевыми полями. Функция подсчитывает уникальные непустые значения, что идеально для метрик типа “количество уникальных клиентов”. Но помните: DISTINCTCOUNT(столбец) может дать результат меньше, чем количество строк в справочнике, если есть неиспользуемые значения.
```

Сравнение с альтернативами

В Excel функции СУММ, СРЗНАЧ, МИН, МАКС работают с диапазонами ячеек, которые вы явно указываете. Изменение диапазона требует редактирования формулы. В DAX агрегаты работают со столбцами, а фильтрация происходит автоматически через контекст. Одна мера DAX заменяет десятки формул Excel для разных срезов данных.

SQL использует агрегатные функции с GROUP BY для группировки данных. В DAX группировка происходит автоматически через структуру отчета — строки и столбцы сводной таблицы. Это упрощает синтаксис, но требует понимания контекста. SQL дает больше контроля над группировкой, DAX — больше гибкости в использовании.

Power Query также имеет функции агрегации при группировке данных. Но агрегаты Power Query вычисляются при загрузке и сохраняются статически. Агрегаты DAX динамические — пересчитываются при каждом изменении фильтров. Используйте Power Query для предварительной агрегации больших объемов, DAX — для интерактивной аналитики.

Частые ошибки

Использование COUNT вместо COUNTA для текстовых полей — классическая ошибка. COUNT игнорирует текст и вернет 0, даже если столбец полон данных. Всегда проверяйте тип данных столбца перед выбором функции подсчета. Для универсального подсчета непустых значений используйте COUNTA.

Игнорирование влияния BLANK на AVERAGE приводит к неверным выводам. Если из 10 магазинов только 5 имеют продажи, AVERAGE посчитает среднее по 5 значениям, а не по 10. Это математически корректно, но может не соответствовать бизнес-логике. При необходимости учета нулевых продаж создавайте отдельную логику.

Применение DISTINCTCOUNT к столбцам с высокой кардинальностью может замедлить отчеты. Подсчет уникальных значений в столбце с миллионами уникальных ID требует значительных ресурсов. Оценивайте необходимость таких метрик и при возможности выполняйте предварительную агрегацию.

```dax
Забывание о контексте при создании мер — источник неожиданных результатов. Мера [Total Sales] := SUM(Sales[Amount]) в ячейке сводной таблицы покажет не все продажи, а только те, что соответствуют фильтрам строки и столбца. Это особенность, а не ошибка, но требует привыкания после Excel.
```

Закрепление материала

Контрольные вопросы

Какая функция подсчитает количество непустых текстовых значений в столбце?

Что вернет функция AVERAGE, если половина значений в столбце - BLANK?

Можно ли использовать MIN и MAX с датами?

В чем разница между COUNT и COUNTROWS?

Как DISTINCTCOUNT обрабатывает повторяющиеся значения?

Что произойдет при делении результата SUM на результат COUNT, если COUNT вернул 0?

Какая функция быстрее работает на больших данных - SUM или DISTINCTCOUNT?

Ответы

COUNTA подсчитывает непустые значения любого типа включая текст, COUNT работает только с числами и датами.

AVERAGE вычислит среднее только по непустым значениям, BLANK-значения полностью игнорируются при подсчете.

Да, MIN вернет самую раннюю дату, MAX - самую позднюю, даты сравниваются хронологически.

COUNT считает непустые значения в столбце, COUNTROWS считает все строки таблицы независимо от содержимого.

DISTINCTCOUNT возвращает количество уникальных значений, каждое значение учитывается только один раз.

Деление на ноль в DAX возвращает BLANK, а не ошибку, что безопаснее для отчетов.

SUM значительно быстрее, так как простое сложение, DISTINCTCOUNT требует отслеживания уникальности.
