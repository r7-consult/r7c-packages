# Урок 27. Практика: Изучение метода `Api.AddComment` по эталону Р7

Вы прочитали шаблон `AddComment_macroR7.js` (Урок 26). Теперь мы отработаем его на практике. 
В вашей корпоративной системе документооборота каждый файл проходит "чек-лист" администратора-бота. Вместо того, чтобы разукрашивать ячейки желтым цветом (Урок 16), скрипт может оставлять невидимые, но кликабельные системные **Комментарии (Comments / Notes)** со статусами проверки (Например: "Данные валидны", "Не совпадает ИНН").

## 🎯 Архитектура Комментариев в Р7-Офис

Во многих системах комментарии привязываются строго к ячейке (`Range("A1").AddComment("...")`).
Но в Р7-Офис архитекторы сделали глобальный инструмент потока (Discussion thread). 
Метод `Api.AddComment("Текст_Комментария", "Имя_Автора_Необязательно")` работает на уровне глобального `API`. Комментарий просто падает в документ, как в чат.

И чтобы прочитать все комментарии книги, вы делаете запрос к "АПИ-базе данных чата":
```javascript
let myComments = Api.GetComments(); 
```
Вы получаете массив объектов (Collection).

## 📝 Постановка задачи

В заготовке кода вы уже видите архитектурную броню: `'use strict'`, `try/catch`, проверку на существование `Api`. 
Вам нужно дописать только Ядро (внутри секции `try`):

1. Вызовите функцию `Api.AddComment("Проверить эту ячейку", "System");`. 
2. Вызовите функцию `Api.AddComment("Одобрено");`. В этот раз передайте только 1 параметр (текст).
3. Инстанциируйте запрос комментариев из движка: сохраните массив в переменную `let allComments = Api.GetComments();`.
4. Возьмите первый элемент "чата" (под индексом `0`!). Вызовите у него метод `.GetText()` и отправьте готовый текст (Конкатенацию 'Текст: ' + текст_комментария) в ячейку `A1` на `worksheet`. 

## 🧠 Глубинка API

Почему мы пишем `Api.AddComment`, а не `sheet.AddComment`?
Потому что комментарии, макросы, стили шрифтов и защита (Protection) являются глобальными свойствами "Рабочего Пространства" (Workspace), а не отдельного листа. Разработчики шаблонов из *R7-Consult* гениально подчеркивают это в своем коде. Если вы понимаете такие мелочи, вы уже мыслите как `Middle+ Developer`.
