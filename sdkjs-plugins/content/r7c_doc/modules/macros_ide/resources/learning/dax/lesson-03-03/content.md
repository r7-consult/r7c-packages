# Урок 11. CALCULATE — сердце DAX

Вступление

Представьте волшебника, способного изменять реальность одним заклинанием. Он может сделать невидимое видимым, заставить время течь иначе, создать параллельные миры с другими законами. В мире DAX таким волшебником является функция CALCULATE. Она может игнорировать фильтры отчета, создавать собственные правила фильтрации, вычислять значения в альтернативной реальности. Без CALCULATE невозможны сравнения периодов, расчет долей от общего, создание накопительных итогов.

CALCULATE — это не просто функция, это философия DAX. Она позволяет программно управлять фильтровым контекстом, который до сих пор создавался только структурой отчета. С CALCULATE вы становитесь архитектором контекстов, создавая сложнейшие аналитические сценарии в одной формуле. Это функция, отличающая новичка от профессионала в DAX.

В предыдущих уроках мы изучили, как работают контексты и как они влияют на вычисления. Теперь научимся управлять ими программно, открывая безграничные возможности анализа.

Теоретическая часть

Основные понятия

```dax
CALCULATE — это функция модификации фильтрового контекста, позволяющая изменить условия вычисления выражения. CALCULATE принимает выражение для вычисления и набор модификаторов контекста, создавая новый контекст специально для этого вычисления. Представьте CALCULATE как временную реальность, где действуют особые правила фильтрации. Например, CALCULATE(SUM(Sales[Amount]), Product[Color] = “Red”) вычислит сумму продаж только для красных товаров, независимо от текущих фильтров отчета по цвету.
```

Модификатор фильтра — это аргумент CALCULATE, изменяющий фильтровый контекст. Модификаторы могут добавлять новые фильтры, удалять существующие или заменять их. Существует три типа модификаторов: булевые выражения (Product[Color] = “Red”), табличные функции (ALL(Product)), и специальные функции модификации (KEEPFILTERS, REMOVEFILTERS). Каждый модификатор применяется к контексту последовательно, создавая финальный контекст вычисления.

```dax
ALL — функция удаления фильтров, возвращающая все строки таблицы или столбца, игнорируя текущие фильтры. ALL(Product) вернет все продукты, даже если в отчете выбрана конкретная категория. ALL можно применить к столбцу: ALL(Product[Color]) удалит только фильтр по цвету, сохранив остальные фильтры таблицы Product. Это ключевой инструмент для создания метрик, независимых от определенных измерений.
KEEPFILTERS — функция сохранения существующих фильтров при добавлении новых. По умолчанию CALCULATE заменяет фильтр столбца при указании нового условия. KEEPFILTERS создает пересечение нового и существующего фильтров. Например, если в отчете выбран цвет “Blue”, то CALCULATE(SUM(Sales[Amount]), KEEPFILTERS(Product[Color] = “Red”)) вернет пустой результат, так как товар не может быть одновременно красным и синим.
```

Ключевая механика

Механизм работы CALCULATE можно представить как конструктор альтернативной реальности. Сначала CALCULATE копирует текущий фильтровый контекст. Затем последовательно применяет каждый модификатор к этой копии. После применения всех модификаторов, выражение вычисляется в новом контексте. Оригинальный контекст остается неизменным — CALCULATE создает изолированное пространство вычисления.

Представьте CALCULATE как фотошоп для данных. Вы берете исходную фотографию (текущий контекст), применяете фильтры и эффекты (модификаторы), получаете модифицированное изображение (результат). Исходная фотография не меняется, вы работаете с копией. Можно применить несколько эффектов последовательно, каждый изменяет результат предыдущего.

Порядок применения модификаторов критически важен. Сначала применяются функции удаления фильтров (ALL, REMOVEFILTERS), затем добавляются новые фильтры. Если указано несколько условий для одного столбца без KEEPFILTERS, применяется только последнее. Это поведение часто приводит к неожиданным результатам у новичков.

Контекстный переход в CALCULATE происходит автоматически при вызове из строкового контекста. Если CALCULATE используется в вычисляемом столбце или итераторе, текущая строка преобразуется в набор фильтров. Это мощный механизм, но может существенно замедлить вычисления при большом количестве уникальных комбинаций.

Вложенные CALCULATE работают последовательно, каждый создает свой уровень модификации. Внутренний CALCULATE получает контекст от внешнего и модифицирует его дальше. Это позволяет создавать сложные многоуровневые вычисления, но усложняет понимание и отладку формул.

Синтаксис и правила

Базовый синтаксис CALCULATE: CALCULATE(expression, filter1, filter2, …). Первый параметр обязателен — это выражение для вычисления. Далее идут опциональные модификаторы фильтров, их может быть от нуля до множества. Каждый модификатор применяется к контексту. Пример: CALCULATE(SUM(Sales[Amount]), Product[Category] = “Bikes”, Customer[Country] = “USA”).

```dax
ALL имеет несколько вариантов использования. ALL(Table) удаляет все фильтры таблицы. ALL(Table[Column]) удаляет фильтр только указанного столбца. ALL(Table[Column1], Table[Column2]) удаляет фильтры нескольких столбцов. Важно: ALL возвращает таблицу, поэтому часто используется внутри CALCULATE как модификатор.
REMOVEFILTERS — синоним ALL, введенный для ясности кода. REMOVEFILTERS(Product) эквивалентно ALL(Product), но явно показывает намерение удалить фильтры. ALLEXCEPT удаляет все фильтры таблицы, кроме указанных столбцов: ALLEXCEPT(Product, Product[Category]) сохранит только фильтр по категории.
KEEPFILTERS оборачивает условие фильтра для сохранения существующих фильтров: CALCULATE(SUM(Sales[Amount]), KEEPFILTERS(Product[Color] = “Red”)). Без KEEPFILTERS условие Product[Color] = “Red” заменит любой существующий фильтр по цвету. С KEEPFILTERS создается пересечение фильтров.
```

Сравнение с альтернативами

В Excel для изменения условий расчета приходится создавать отдельные формулы с разными условиями SUMIF, COUNTIF. Каждая комбинация условий требует новой формулы. CALCULATE позволяет динамически менять условия в одной формуле, адаптируясь к контексту и добавляя собственные правила.

SQL использует подзапросы и оконные функции для похожих задач. Но SQL-запрос фиксирован при написании, а CALCULATE динамически адаптируется к контексту отчета. Это делает DAX идеальным для интерактивной аналитики, где пользователи постоянно меняют фильтры.

Power Query может фильтровать данные при загрузке, но это статическая фильтрация. Изменение условий требует обновления запроса и перезагрузки данных. CALCULATE работает на лету, мгновенно пересчитывая результаты при изменении условий.

Частые ошибки

Забывание о замене фильтров по умолчанию — самая распространенная ошибка. Новички думают, что CALCULATE(SUM(Sales[Amount]), Product[Color] = “Red”) добавит фильтр к существующим. В реальности он заменит любой текущий фильтр по цвету. Используйте KEEPFILTERS для пересечения фильтров.

Неправильное использование ALL с несколькими столбцами создает неожиданные результаты. ALL(Product[Category], Product[Color]) удаляет фильтры обоих столбцов одновременно, а не каждого независимо. Для независимого удаления используйте отдельные ALL в разных CALCULATE.

Избыточное вложение CALCULATE усложняет формулы без необходимости. Часто можно объединить модификаторы в одном CALCULATE вместо создания вложенных конструкций. Это улучшает читаемость и производительность.

Игнорирование контекстного перехода в CALCULATE приводит к медленным формулам. При использовании в итераторах CALCULATE выполняет переход для каждой строки, что может быть очень затратно. Оптимизируйте такие сценарии.

Закрепление материала

Контрольные вопросы

Что делает функция CALCULATE с фильтровым контекстом?

Как ALL влияет на существующие фильтры?

В чем разница между ALL и ALLEXCEPT?

Когда нужно использовать KEEPFILTERS?

Что происходит при указании нескольких фильтров в CALCULATE?

Как CALCULATE работает в строковом контексте?

Можно ли использовать CALCULATE без модификаторов фильтра?

Ответы

CALCULATE создает копию текущего фильтрового контекста и модифицирует ее согласно указанным условиям перед вычислением выражения.

ALL удаляет все фильтры с указанной таблицы или столбцов, возвращая полный набор данных для вычислений.

ALL удаляет все фильтры, ALLEXCEPT удаляет все фильтры кроме указанных столбцов, сохраняя выбранные измерения.

KEEPFILTERS нужен когда требуется пересечение нового фильтра с существующим, а не замена существующего фильтра.

Множественные фильтры применяются через AND, но фильтры одного столбца без KEEPFILTERS заменяют друг друга.

В строковом контексте CALCULATE автоматически выполняет переход контекста, превращая текущую строку в набор фильтров.

Да, CALCULATE без модификаторов выполняет только переход контекста, что полезно в вычисляемых столбцах.
