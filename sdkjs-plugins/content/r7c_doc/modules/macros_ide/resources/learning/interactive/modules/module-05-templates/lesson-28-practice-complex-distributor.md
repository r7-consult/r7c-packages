# Урок 28. Практика Архитектора: Конструируем алгоритм Round-Robin

Как разработчик, пришедший из мира Excel/VBA, вы прекрасно понимаете разницу между медленным обращением к COM-объекту (DOM) в цикле и быстрой работой в оперативной памяти с последующей массовой выгрузкой (Batch Write).

В VBA мы часто формировали 2D-массив (вариантный массив) и выплевывали его на лист: `Range("A1").Resize(UBound(arr), 2).Value = arr`.
В JavaScript API Р7-Офис подход абсолютно идентичен: формирование массива массивов `[[]]` и один вызов метода `.SetValue(arr)`.

## 🎯 Бизнес-задача: Балансировщик нагрузки

Менеджеру нужно раскидать 1000 "холодных" контактов (строк) между 3 операторами колл-центра, распределяя их равномерно (Иван, Анна, Петр, Иван, Анна...). Это классический алгоритм кольцевого буфера (**Round-Robin**).

Справа дан стартовый код. В оперативной памяти загружены 10 задач и 3 менеджера. 

**Ваша задача:**
1. Написать цикл сборки 2D-массива `finalDistribution`. 
2. Управлять счетчиком `mIndex`, сбрасывая его в `0` при достижении длины массива `managers` (реализация классического кольца).
3. Очистить полигон вывода на листе.
4. Вычислить динамический адрес диапазона (`"A1:B" + длина_массива`) и произвести единовременную выгрузку через `SetValue`.

*Обратите внимание: никаких детских `sheet.GetRange("A" + i).SetValue(...)` в цикле. Мы пишем высоконагруженный Production-код.*

## ✍️ Эталон алгоритма (Для самопроверки)

В JavaScript работа со сбросом счетчика выглядит невероятно лаконично:

```javascript
for (let i = 0; i < tasks.length; i++) {
    // Формируем одномерный массив (строку) и кидаем в матрицу
    finalDistribution.push([tasks[i], managers[mIndex]]);
    
    mIndex++;
    if (mIndex === managers.length) { 
        mIndex = 0; // Кольцевой сброс
    }
    
    // Альтернативный "хакерский" вариант сброса без IF (через Modulo):
    // mIndex = (mIndex + 1) % managers.length;
}

sheet.GetRange("A1:B100").Clear();
let addr = "A1:B" + finalDistribution.length;
sheet.GetRange(addr).SetValue(finalDistribution);
```

Запустите код кнопкой (Run). Если автотесты загорятся зеленым — вы успешно интегрировали серверный алгоритм балансировки в табличный процессор Р7-Офис.
