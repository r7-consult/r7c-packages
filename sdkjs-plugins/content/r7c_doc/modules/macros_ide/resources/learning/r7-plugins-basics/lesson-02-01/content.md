**Урок 2.1. Минимальный состав плагина: config.json, index.html, code.js**

Любой плагин для табличного редактора **Р7-Офис** представляет собой архив с расширением .plugin, содержащий набор обязательных и опциональных файлов. Минимально необходимый набор включает три компонента:

-    **config.json** — конфигурационный файл, описывающий метаданные плагина: имя, уникальный идентификатор (GUID), версию, поддерживаемые типы редакторов и путь к HTML-интерфейсу.
-    **index.html** — файл пользовательского интерфейса, отображаемый при запуске плагина. Может содержать кнопки, формы, текст и ссылки на скрипты.
-    **code.js** (или любой другой .js-файл) — файл с JavaScript-логикой, реализующий взаимодействие с API Р7 через функции window.Asc.plugin.init() и window.Asc.plugin.button().

Без этих трёх файлов плагин не будет корректно распознан системой. Все они должны находиться в корне архива, а их имена и структура должны строго соответствовать требованиям платформы.
