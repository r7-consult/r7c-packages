Бюджеты планируются и контролируются нарастающим итогом. Без специализированных функций пришлось бы создавать сложные конструкции с множественными условиями и ручным определением границ периодов.

Теоретическая часть

Основные понятия

```dax
TOTALYTD (Total Year-to-Date) — функция расчета накопительного итога с начала года до текущей даты в контексте. TOTALYTD автоматически определяет начало года и суммирует все значения от 1 января до последней даты в текущем фильтровом контексте. Например, если выбран март, функция вернет сумму за январь, февраль и март. TOTALYTD(SUM(Sales[Amount]), Calendar[Date]) рассчитает продажи с начала года.
DATESYTD (Dates Year-to-Date) — функция, возвращающая все даты с начала года до текущей даты контекста. В отличие от TOTALYTD, которая выполняет агрегацию, DATESYTD возвращает набор дат для использования в CALCULATE. Это дает больше гибкости для создания сложных накопительных расчетов. DATESYTD можно комбинировать с любыми агрегатными функциями.
```

Финансовый календарь — это календарь с годом, начинающимся не 1 января. Многие компании используют финансовые годы: апрель-март, июль-июнь, октябрь-сентябрь. Функции YTD поддерживают финансовые календари через опциональный параметр year_end_date. TOTALYTD(SUM(Sales[Amount]), Calendar[Date], “6/30”) будет считать финансовый год, заканчивающийся 30 июня.

Накопительный итог (Running Total) — это сумма значений от начальной точки до текущей позиции. В контексте времени это обычно означает суммирование от начала периода (года, квартала, месяца) до текущей даты. Накопительные итоги показывают прогресс и тренд, в отличие от точечных значений.

Ключевая механика

Механизм работы функций период-до-даты основан на динамическом определении границ периода и фильтрации дат. Функция анализирует текущий контекст, определяет последнюю видимую дату, находит начало соответствующего периода (год, квартал, месяц), создает набор всех дат от начала до конца, применяет этот набор как фильтр для расчета.

Представьте накопительные функции как заполнение стакана водой. TOTALYTD начинает наливать воду с 1 января (или начала финансового года) и останавливается на текущей дате. Уровень воды — это накопленное значение. TOTALQTD начинает с начала квартала — это как если бы стакан опустошали каждые три месяца и начинали заполнять заново. TOTALMTD опустошает стакан каждый месяц.

Адаптация к контексту делает эти функции универсальными. В годовом отчете TOTALYTD покажет полный год (так как конец года — это последняя дата). В месячном отчете — накопление до конца выбранного месяца. В дневном — точное накопление до выбранного дня. Одна формула работает на всех уровнях детализации.

Обработка неполных периодов — важная особенность. Если текущий месяц не закончен, TOTALMTD покажет накопление до последней даты с данными, а не до конца месяца. Это обеспечивает корректное сравнение: накопление за 15 дней текущего месяца можно сравнить с накоплением за 15 дней прошлого месяца.

Финансовые календари требуют особого внимания. При указании конца финансового года функции автоматически пересчитывают все границы. Финансовый год, заканчивающийся 30 июня, начинается 1 июля предыдущего календарного года. Q1 финансового года — это июль-сентябрь, а не январь-март.

Синтаксис и правила

```dax
TOTALYTD имеет три параметра: TOTALYTD(expression, dates, [year_end_date], [filter]). Первые два обязательны — выражение для накопления и столбец дат. Опциональный year_end_date задает конец финансового года. Последний параметр filter позволяет добавить дополнительные условия:
[Sales YTD] :=
TOTALYTD(
SUM(FactInternetSales[SalesAmount]),
'DimDate'[FullDateAlternateKey],
"12/31"  // опционально, по умолчанию 31 декабря
)
DATESYTD возвращает таблицу дат и имеет два параметра: DATESYTD(dates, [year_end_date]). Используется внутри CALCULATE для большей гибкости:
[Custom YTD] :=
CALCULATE(
AVERAGE(FactInternetSales[SalesAmount]),
DATESYTD('DimDate'[FullDateAlternateKey])
)
```

Квартальные и месячные варианты работают аналогично. TOTALQTD и TOTALMTD не имеют параметра year_end_date, так как кварталы и месяцы универсальны:

```dax
[Sales QTD] := TOTALQTD(SUM(FactInternetSales[SalesAmount]), 'DimDate'[FullDateAlternateKey])
[Sales MTD] := TOTALMTD(SUM(FactInternetSales[SalesAmount]), 'DimDate'[FullDateAlternateKey])
```

Сравнение с альтернативами

В Excel для накопительных итогов используются формулы с расширяющимися диапазонами ($A$1:A10) или СУММЕСЛИМН с условиями на даты. Это требует отдельных формул для каждой строки и ручной настройки границ. DAX автоматизирует процесс одной универсальной формулой.

SQL подходы включают оконные функции с SUM() OVER (ORDER BY date ROWS UNBOUNDED PRECEDING) или самосоединения с условиями на даты. SQL эффективен для статических отчетов, но не адаптируется к изменению фильтров. DAX динамически пересчитывает накопление при любом изменении контекста.

Power Query может предрассчитать накопительные итоги при загрузке, но это увеличивает объем данных и снижает гибкость. Изменение логики накопления требует перезагрузки. DAX вычисляет на лету без дублирования данных.

Частые ошибки

Использование YTD-функций без полного календаря приводит к некорректным результатам. Если в календаре есть пропуски, накопление может прерваться или дать неверные значения. Всегда используйте непрерывный календарь с полными годами.

Путаница с финансовыми календарями при указании year_end_date. Параметр “6/30” означает, что год заканчивается 30 июня, а не начинается. Финансовый год 2014, заканчивающийся 30 июня 2014, начался 1 июля 2013. Проверяйте границы периодов при настройке.

Неправильное использование DATESYTD вместо TOTALYTD. DATESYTD возвращает даты, а не сумму. Для простого накопления используйте TOTALYTD. DATESYTD нужен только для сложных сценариев с кастомной агрегацией.

Забывание о влиянии других фильтров на YTD-расчеты. Если выбран фильтр по продукту, YTD покажет накопление только для этого продукта. Это корректное поведение, но может удивить при первом использовании.

## DAX-решение

```dax
// Продажи за период
[Sales Amount] :=
SUM(FactInternetSales[SalesAmount])
// Накопительные продажи с начала года
[Sales YTD] :=
TOTALYTD(
[Sales Amount],
'DimDate'[FullDateAlternateKey]  // столбец дат для YTD

DIVIDE(
[Sales YTD],
```

Ответы

TOTALYTD рассчитывает накопительный итог с начала года до текущей даты в контексте.

TOTALYTD выполняет агрегацию и возвращает значение, DATESYTD возвращает набор дат для использования в CALCULATE.

Через опциональный параметр year_end_date, например “6/30” для года, заканчивающегося 30 июня.

TOTALQTD сбрасывает накопление в начале каждого квартала (1 января, 1 апреля, 1 июля, 1 октября).

TOTALMTD покажет накопление с 1 числа по 15 число текущего месяца.

Да, YTD-функции требуют непрерывный календарь без пропусков для корректной работы.

YTD-функции учитывают все активные фильтры, показывая накопление только для отфильтрованных данных.
