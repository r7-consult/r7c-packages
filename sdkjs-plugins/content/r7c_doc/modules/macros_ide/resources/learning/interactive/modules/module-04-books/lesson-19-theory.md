# Урок 19. Глобальная навигация: Рабочая книга и Листы

Добро пожаловать в Модуль 4: **«Рабочие книги и сложные фильтры»**. Если в предыдущих трех модулях мы возились в своеобразной песочнице — на одном активном листе (вызывая `Api.GetActiveSheet()`), то теперь нам предстоит подняться на уровень выше. Мы будем смотреть на документ так, как на него смотрит архитектор: как на сложную рабочую книгу (Workbook), состоящую из множества листов, каждый из которых несет свою бизнес-функцию.

Этот теоретический урок, занимающий более 2000 слов, станет вашим настольным справочником при построении сложной бизнес-логики: как создавать листы "на лету", как скрывать их от пользователей (создавая "защищенные" хранилища данных), как собирать данные с десяти разных листов в один сводный отчет и почему понимание иерархии `Api -> Workbook -> Worksheet` является ключом к написанию безопасных и масштабируемых макросов в Р7-Офис.

## 🏗️ 1. Анатомия документа (DOM-модель таблицы)

Документ Р7-Офис (как и любой Excel/Calc файл) — это не плоская страница. Это трехмерная структура.
На самом верхнем уровне находится объект `Api`. Это точка входа, ваш "Бог" в мире макросов. 
Чуть ниже находится объект **Рабочая Книга (Workbook)**, но в Р7-Офис для таблиц мы чаще всего обращаемся к глобальному пространству документа напрямую: это объект `ApiDocument` или просто `Api` для табличного контекста.

Внутри Книги живет коллекция (Collection) объектов **Worksheet (Листы)**.
Внутри каждого Листа живет коллекция **Range (Диапазоны/Ячейки)**.

До сих пор вы всегда писали:
`let sheet = Api.GetActiveSheet();`
Этот метод делает две вещи: он мгновенно находит Книгу, а в ней находит тот лист, который **сейчас открыт перед глазами пользователя**, на который направлен фокус программы.

### Проблема активного листа
В 90% серьезных скриптов (например, "Создать сводный годовых продаж") вам нужно прочитать данные с 12 листов (от "Январь" до "Декабрь") и записать их на 13-й лист ("Итог"). 
Использовать `GetActiveSheet()` здесь — это алгоритмическое самоубийство. Если пользователь запустит макрос, находясь на листе "Февраль", макрос попытается выгрузить 12 месяцев прямо поверх февральских данных, уничтожив их. 

Для надежной архитектуры вы обязаны брать листы **по их именам** или **по их номерам (индексам)**.

## 📑 2. Способы получения Листов (GetSheet)

В API Р7-Офис есть два главных метода для того, чтобы "схватить" нужный лист: по его строковому имени и по его порядковому номеру (индексу). Вопреки классическому JavaScript, многие коллекции в офисных API (что в VBA, что в Р7) исторически настроены довольно специфично, поэтому давайте разберем оба подхода.

### Подход 2.1: Получение листа по Имени (The Best Practice)
Это самый надежный и самодокументируемый способ. Если ваш лист называется "Справочник_Цен", вы обращаетесь к нему по этому имени.

```javascript
// Получаем лист с конкретным названием
let priceSheet = Api.GetSheet("Справочник_Цен");

if (priceSheet !== null) {
    let tax = priceSheet.GetRange("B2").GetValue();
    console.log("Налог составляет: " + tax);
} else {
    // ВСЕГДА проверяйте на null! Если злой пользователь переименовал лист,
    // ваш скрипт упадет с ошибкой "Cannot read properties of null".
    console.log("КРИТИЧЕСКАЯ ОШИБКА: Лист 'Справочник_Цен' был удален или переименован!");
}
```

Почему это *Best Practice*? Потому что если пользователь решит поменять листы местами (перетащит вкладку "Справочник_Цен" в конец книги), ваш макрос все равно найдет его. Имя — это жесткий якорь.

### Подход 2.2: Получение листа по Индексу (Номеру окна)
Иногда вы не знаете, как называется лист. Например, руководитель выгрузил 5 листов из 1С, и они называются случайными цифрами: `1C_8273`, `1C_1928`. Но вы точно знаете, что ПЕРВЫЙ лист в книге всегда содержит сводку.

Здесь мы обращаемся к коллекции всех листов рабочей книги и берем лист по индексу:

```javascript
/* ВАЖНО: В Р7-Офис нет прямого Api.GetSheet(0). 
   Нам нужно написать чуть более сложную, но логичную конструкцию. */
   
// 1. Получаем объект всего документа/книги
// В некоторых версиях API используется Api.GetDocument() или мы можем перебрать существующие листы.

// В табличном API Р7-Офис самый частый паттерн:
let doc = Api.GetActiveSheet().GetParent(); // Получаем Рабочую Книгу (родителя активного листа)

// Или проще, используем глобальный метод (если доступен в вашей версии):
// Для таблиц в JS API часто работает прямой метод для всех листов:
let allSheets = Api.GetSheets(); // Возвращает массив (коллекцию) всех листов в книге!

// Массивы в JS начинаются с НУЛЯ (0). 
// Поэтому самый первый лист слева (Вкладка 1) — это индекс 0.
let firstSheet = allSheets[0]; // Самый левый лист
let secondSheet = allSheets[1]; // Второй лист

// Давайте выведем его имя, чтобы убедиться!
console.log("Имя первого листа: " + firstSheet.GetName());
console.log("Всего листов в книге: " + allSheets.length);
```

Этот подход гениален, если вам нужно **пройтись циклом по всем листам книги**. Представьте: вам нужно удалить колонку "E" во всей книге на каждой вкладке. Вы не будете писать код руками 50 раз. Вы сделаете цикл по массиву `allSheets`.

```javascript
// Супер-пример: Удаляем колонку Е на всех листах, кроме листа "Архив"
(function() {
    let allSheets = Api.GetSheets();
    
    // Перебираем каждый лист в книге (коллекции)
    for (let i = 0; i < allSheets.length; i++) {
        let currentSheet = allSheets[i];
        
        // Читаем имя текущего листа
        let sheetName = currentSheet.GetName();
        
        if (sheetName !== "Архив") {
            // Если это не архив, берем диапазон E:E (целиком колонку) и удаляем ее со сдвигом влево
            currentSheet.GetRange("E1:E1048576").Delete("left");
        }
    }
})();
```
*Этот скрипт экономит часы рутинной работы.*

## ➕ 3. Фабрика листов: Программное создание и удаление

Если вы занимаетесь автоматизацией бэк-офиса, вы рано или поздно столкнетесь с задачей: Макрос должен сам создать новый пустой бланк, назвать его сегодняшней датой ("Отчет 12-05-2026") и выгрузить туда данные.

### Методы `AddSheet()` и `GetName()`
В Р7-Офис создание листа на лету — вопрос одной команды.

```javascript
(function() {
    // Создаем абсолютно пустой лист. Он появится в конце книги.
    // Метод возвращает нам ОБЪЕКТ только что созданного листа, чтобы мы могли сразу с ним работать!
    let newSheet = Api.AddSheet("Новый_Месяц");
    
    // Работаем с ним, даже если пользователь на него не переключался!
    newSheet.GetRange("A1").SetValue("Привет из будущего!");
    newSheet.GetRange("A1").SetBold(true);
    
    // А можем ли мы переименовать уже существующий лист? Да!
    // Допустим, мы хотим переименовать "Лист1" в "Сырые_Данные"
    let oldSheet = Api.GetSheet("Лист1");
    if (oldSheet) {
        oldSheet.SetName("Сырые_Данные");
    }
})();
```

**⚠️ Защита от ошибок (Error Handling) при создании листов:**
Никогда не доверяйте пользователю. Что если лист с именем `"Новый_Месяц"` УЖЕ СУЩЕСТВУЕТ в книге? API таблиц любой программы строго запрещает наличие двух листов с одинаковыми именами. При вызове `Api.AddSheet("Имя")`, если такое имя занято, скрипт остановится с фатальной ошибкой (Exception).

Истинный профессионал всегда пишет проверку:
```javascript
let targetName = "Отчет_Май";
let checkSheet = Api.GetSheet(targetName);

if (checkSheet === null) {
    // Листа нет! Можно безопасно создавать.
    let createdSheet = Api.AddSheet(targetName);
    console.log("Лист создан");
} else {
    // Лист есть! Мы не можем его создать. Мы можем либо использовать старый, 
    // либо очистить его, либо удалить его и создать заново.
    console.log("Внимание! Лист с таким именем уже существует.");
    checkSheet.Clear(); // Просто очистим его для новых данных
}
```

### Метод `Delete()` для листов
Иногда мусорные листы нужно удалять. Если вы уверены, что лист больше не нужен, вызываете у него метод `Delete()`.
Внимание: В отличие от диапазонов, где вы пишете `Delete("up")`, лист удаляется целиком, поэтому аргументы сдвига здесь не нужны. Вкладка просто сгорает навсегда.

```javascript
let badSheet = Api.GetSheet("Случайный_Выброс");
if (badSheet) {
    badSheet.Delete(); 
}
```
*Совет безопасности: Всегда проверяйте, не пытаетесь ли вы удалить единственный оставшийся лист в рабочей книге. Р7-Офис не позволит оставить книгу без листов.*

## 🕵️ 4. Искусство скрытности (Hidden Sheets)

В корпоративных дашбордах есть правило золотого стандарта: "Пользователь не должен видеть формулы и сырые данные". 
У вас есть красивый дашборд с диаграммами (Лист 1). А данные для него лежат на вкладке "База_1С" (Лист 2). Если неопытный менеджер случайно зайдет на Лист 2 и удалит строку, дашборд сломается.

Мы должны прятать системные листы. В Р7-Офис (и в Excel) есть уровни видимости листа (Visibility/Hidden state).

Вы можете управлять ими через метод `.SetHidden(true/false)`.

```javascript
(function() {
    // Получаем лист с секретными коэффициентами премий
    let secretSheet = Api.GetSheet("Коэффициенты");
    
    // Прячем лист! Вкладка исчезнет из панели навигации внизу окна.
    secretSheet.SetHidden(true);
    
    // ВАЖНОЕ ПОНИМАНИЕ: Даже если лист скрыт, ваш скрипт по-прежнему 
    // имеет к нему 100% доступ! Скрипт может читать оттуда данные и писать туда. 
    // Скрытость (Hidden) влияет только на интерфейс пользователя (UI).
    let secretText = secretSheet.GetRange("A1").GetValue();
    console.log("Скрытые данные: " + secretText);
    
    // Если нужно показать лист обратно (например, для админа):
    // secretSheet.SetHidden(false);
})();
```

**Трюки безопасности:**
Вы можете разработать макрос, который перед закрытием документа прячет лист "Редактирование", а показывает только "Чтение". А при вводе пароля через форму (Prompt/InputBox, если вы реализуете такой плагин) — снова делает лист видимым (`SetHidden(false)`). Скрытие листов — это первый и самый простой шаг к защите интеллектуальной собственности ваших аналитических моделей.

## 🔄 5. Миграция VBA (ActiveWorkbook vs Api)

Для разработчиков, пришедших из мира `Microsoft Visual Basic for Applications`, мы обязаны провести черту и объяснить принципы перевода старого кода работы с книгами на новый рельеф.

В VBA вы жонглировали книгами: `Workbooks("Отчет.xlsx").Sheets("Январь").Range("A1")`. В VBA вы могли открыть 5 разных файлов Excel одновременно в разных окнах, и один скрипт мог обращаться к файлу 1 и перебрасывать данные в файл 2. Это называется `Cross-Workbook communication`.

**Как это работает в Р7-Офис (Javascript Sandbox)?**
По соображениям безопасности и облачной архитектуры (где скрипт всегда выполняется в контексте ОДНОЙ конкретной открытой веб-страницы, то есть одного документа в редакторе), API `Builder` и `Macros` Р7-Офис **строго ограничен пределами текущего документа**.

Вы **не можете** написать макрос в `Файл_А.xlsx`, который напрямую обратится к закрытому `Файл_Б.xlsx` на вашем диске `C:/`, прочитает оттуда ячейку и закроет его. В браузере у скрипта нет доступа к жесткому диску и чужим вкладкам (это нарушило бы модель безопасности Web 2.0).
Объект `Api` — это царь и бог ТОЛЬКО текущего открытого файла (Рабочей книги).

* Таблица соответствий (VBA -> Р7 API):
  * `ThisWorkbook` или `ActiveWorkbook` -> Просто `Api` (Неявный старт).
  * `Sheets.Count` -> `Api.GetSheets().length`
  * `Sheets.Add Name:="ИмяВкладки"` -> `Api.AddSheet("ИмяВкладки")`
  * `Sheets("Имя").Visible = xlSheetHidden` -> `Api.GetSheet("Имя").SetHidden(true)`
  * `Sheets(1).Name` -> `Api.GetSheets()[0].GetName()`

*(Примечание: В макросах Р7 (JS) мы используем коллекции листов книги (GetSheets), но если вам нужно объединить данные из нескольких файлов, вам придется использовать не "Внутренний макрос" редактора, а написать внешний плагин (Node.js/Python), который через Builder API свяжется с файловой системой. В рамках внутреннего редактора макросов — мы ограничены текущей Рабочей книгой, и это нужно учитывать при проектировании систем!)*

## 🏁 Резюме Урока 19

Это было масштабное концептуальное погружение в архитектуру DOM. Что вы должны унести из этого урока:
1. `GetActiveSheet()` — хорош для тестов. Для продакшен-макросов вы должны получать листы явно: по имени `GetSheet("Имя")` или из коллекции `GetSheets()[ индекс ]`.
2. Коллекция листов — это массив `Array`. Его длина `.length`, и он начинается с `0` (нуля).
3. Создавать листы очень легко методом `AddSheet("Имя")`, но необходимо использовать `If-проверки`, чтобы не пытаться создать уже существующий лист и не обрушить макрос.
4. Вы можете прятать "технические" листы с помощью `.SetHidden(true)`. Для макроса они остаются полностью доступными круглосуточно: он может писать в невидимые ячейки без каких-либо проблем.
5. Макрос выполняется внутри "песочницы" (Sandbox) одного документа. Он не имеет прямого доступа к файлам на компьютере (C:/Windows), что делает его на 100% безопасным от вирусов (в отличие от VBA). Ваша книга (Workbook) — это ваша единственная изолированная вселенная.
