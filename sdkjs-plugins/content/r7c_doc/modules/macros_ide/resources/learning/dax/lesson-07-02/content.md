Теоретическая часть

Основные понятия

ABC-анализ — метод категоризации элементов по их вкладу в общий результат. Группа A обычно содержит 20% элементов, дающих 80% результата (принцип Парето). Группа B — следующие 30% элементов с 15% вклада. Группа C — оставшиеся 50% элементов с 5% вклада. В DAX это реализуется через накопительные проценты и условную логику. ABC помогает фокусироваться на действительно важных элементах.

```dax
SWITCH(TRUE) — конструкция для проверки множественных условий последовательно. В отличие от вложенных IF, SWITCH(TRUE) более читаем и эффективен. SWITCH(TRUE(), condition1, result1, condition2, result2, …, else_result) проверяет условия сверху вниз и возвращает результат для первого истинного условия. Это идеально для создания сегментов с множественными границами.
```

GENERATESERIES — функция создания таблицы с последовательностью чисел. GENERATESERIES(start, stop, [step]) генерирует ряд от start до stop с указанным шагом. Это основа для создания диапазонов бинирования: возрастные группы, ценовые категории, временные интервалы. Виртуальная таблица диапазонов может использоваться для динамической категоризации.

Квантили — это точки, делящие упорядоченное распределение на равные части по количеству элементов. Квартили делят на 4 части (по 25% элементов), децили — на 10 частей (по 10%). В отличие от ABC, где группы имеют разный размер но фиксированный вклад, квантили имеют одинаковый размер независимо от вклада. Это полезно для равномерного распределения нагрузки или ресурсов.

Ключевая механика

Механизм создания ABC-сегментов основан на расчете накопительного процента вклада. Сначала элементы сортируются по убыванию метрики (например, продаж). Затем рассчитывается накопительный процент: первый элемент дает X%, первый плюс второй — Y%, и так далее. Когда накопительный процент достигает 80%, все элементы до этой точки попадают в группу A. Следующие до 95% — группа B, остальные — C.

Представьте ABC-анализ как упаковку чемодана для путешествия. Сначала вы кладете самые важные вещи (группа A) — они занимают основное место. Затем полезные, но не критичные (группа B). В конце — “на всякий случай” (группа C). Если нужно сократить багаж, вы знаете, от чего отказаться в первую очередь.

```dax
SWITCH(TRUE) работает как многоуровневый фильтр на конвейере. Каждый элемент проходит через серию проверок. Первая проверка, которая дает TRUE, определяет категорию элемента. Важен порядок условий — более строгие условия должны идти первыми, иначе элементы попадут не в те категории.
```

Бинирование через GENERATESERIES создает сетку для категоризации непрерывных значений. Например, для возрастов создается серия: 0, 18, 25, 35, 50, 65, 100. Каждый возраст попадает в интервал между соседними значениями. Это превращает непрерывную переменную в дискретные категории, упрощая анализ и визуализацию.

Динамическая адаптация сегментов означает, что границы групп пересчитываются при изменении фильтров. Если выбран конкретный регион, ABC-анализ покажет распределение внутри этого региона. Топ-20% продуктов в одном регионе могут отличаться от топ-20% в другом. Это делает сегментацию контекстно-зависимой и более точной.

Синтаксис и правила

```dax
SWITCH(TRUE) для создания сегментов имеет специфический синтаксис. Первый параметр всегда TRUE(), затем пары условие-результат:
[Customer Segment] :=
SWITCH(
TRUE(),
[Customer Total Sales] > 10000, "Premium",
[Customer Total Sales] > 5000, "Gold",
[Customer Total Sales] > 1000, "Silver",
"Bronze"  // условие по умолчанию
)
```

## GENERATESERIES принимает от двух до трех параметров

```dax
// Создание возрастных групп
Age Ranges =
GENERATESERIES(0, 100, 10)  // 0, 10, 20, ..., 100
```

## Для ABC-анализа используется комбинация накопительных расчетов и SWITCH

```dax
[ABC Category] :=
SWITCH(
TRUE(),
[Cumulative % of Total] <= 0.8, "A",
[Cumulative % of Total] <= 0.95, "B",
"C"
)
```

Сравнение с альтернативами

В Excel для ABC-анализа используются вспомогательные столбцы с формулами СУММЕСЛИ и ВПР для присвоения категорий. Это требует ручной сортировки и пересчета при изменении данных. DAX автоматизирует процесс и динамически адаптируется к фильтрам.

SQL CASE WHEN концептуально похож на SWITCH(TRUE), но применяется при запросе к базе. SQL-сегментация статична, DAX пересчитывает сегменты при каждом изменении контекста, обеспечивая интерактивность анализа.

Python pandas cut() и qcut() предлагают мощные функции бинирования, но требуют предварительной обработки данных. DAX выполняет сегментацию на лету без изменения исходных данных, что критично для интерактивных дашбордов.

Частые ошибки

Неправильный порядок условий в SWITCH(TRUE) приводит к некорректной категоризации. Условия проверяются последовательно, и первое истинное определяет результат. Всегда располагайте более строгие условия первыми: сначала “&gt;10000”, потом “&gt;5000”, потом “&gt;1000”.

Использование фиксированных границ для ABC вместо процентов искажает анализ. ABC должен адаптироваться к данным: 80% от миллиона или от тысячи — разные суммы. Рассчитывайте границы динамически через проценты, а не hardcode значения.

Игнорирование NULL и нулевых значений при сегментации создает неполные группы. Элементы без данных могут исказить распределение или вообще не попасть в категории. Всегда обрабатывайте краевые случаи явно.

Отсутствие валидации полноты сегментации может привести к потере данных. После создания сегментов проверьте, что сумма элементов во всех группах равна общему количеству. Используйте категорию “Другое” или “Не определено” для неклассифицированных элементов.

Ответы

ABC-анализ делит элементы на группы по вкладу: A (80% результата), B (15%), C (5%); размер групп варьируется.

```dax
SWITCH(TRUE) последовательно проверяет условия и возвращает результат для первого истинного условия.
```

GENERATESERIES создает таблицу с последовательностью чисел для определения границ диапазонов и категорий.

ABC фокусируется на вкладе (разный размер групп), квантили — на равном распределении элементов.

Условия проверяются сверху вниз; первое истинное определяет результат, поэтому строгие условия должны быть первыми.

Сегменты пересчитываются динамически при изменении контекста, показывая распределение в текущем фильтре.

Типично: A — 80% вклада, B — следующие 15%, C — оставшиеся 5%; но границы могут корректироваться.
