Вступление

Представьте финансового директора в конце квартала, сравнивающего фактические результаты с планом. Продажи составили $5.2 миллиона при плане $5 миллионов — хорошо, но правильно ли распределены успехи по месяцам? Совпали ли пики продаж с запланированными? Где именно произошли отклонения? В DAX анализ план/факт требует особых техник: выравнивания календарей, использования TREATAS для связывания несвязанных таблиц бюджетов и корректного распределения годовых планов по дням.

План-факт анализ — это основа управления эффективностью в любой компании. Бюджеты планируются на разных уровнях детализации: годовые цели, квартальные вехи, месячные планы. Фактические данные поступают ежедневно. Сопоставление этих разногранулярных данных, расчет отклонений и анализ причин — ключевые задачи финансового контроллинга. DAX предоставляет инструменты для элегантного решения этих задач без дублирования данных.

Теоретическая часть

Основные понятия

План-факт анализ (Budget vs Actual) — это сравнение запланированных показателей с фактически достигнутыми для оценки эффективности и выявления отклонений. План обычно хранится в отдельной таблице с меньшей гранулярностью (месяцы или кварталы), факт — в детальной транзакционной таблице. Ключевая задача — корректно сопоставить данные разной детализации для расчета отклонений на любом уровне.

Выравнивание календарей — процесс синхронизации временных периодов между планом и фактом. Бюджет на март должен с марта, даже если в плане указан только месяц, а в факте — конкретные даты. DAX использует общий календарь для связывания периодов через временные функции и виртуальные связи.

Распределение бюджета (Budget Allocation) — это разбиение укрупненных плановых показателей на более детальные периоды. Годовой план в $12 миллионов может распределяться равномерно по месяцам ($1 миллион) или пропорционально сезонности прошлого года. Правильное распределение критично для корректного ежедневного контроля выполнения плана.

TREATAS для бюджетов — техника создания виртуальной связи между несвязанными таблицами плана и факта. TREATAS применяет фильтры из одной таблицы к другой без физической связи. Это позволяет гибко сопоставлять бюджеты с фактом, даже если структуры таблиц различаются, без изменения модели данных.

Ключевая механика

Механизм сопоставления план-факт основан на создании общего контекста для сравнения. Фильтр по времени применяется и к плановым, и к фактическим данным через общий календарь. Затем соответствующие значения извлекаются и сравниваются. Важно: гранулярность сравнения определяется наименьшим общим уровнем детализации.

Представьте план-факт анализ как сравнение маршрута GPS с реальным путем водителя. План — это оптимальный маршрут с ключевыми точками и ожидаемым временем. Факт — реальная траектория с учетом пробок и объездов. Выравнивание — это наложение реального пути на запланированный для выявления отклонений. TREATAS работает как универсальный переводчик координат между разными картределение бюджета работает как нарезка пирога. Годовой “пирог” бюджета можно разрезать равными кусками (линейное распределение) или пропорционально аппетиту гостей (по паттерну прошлого). Важно, чтобы сумма кусков равнялась целому пирогу — контрольная сумма должна сохраняться при любом распределении.

Обработка отсутствующих данных критична для план-факт анализа. Если план есть, а факта нет — это невыполнение. Если факт есть, а плана нет — это может быть сверхплановый доход или незапланированный расход. DAX должен корректно обрабатывать оба случая, используя COALESCE или условную логику для замены BLANK значений нулями где необходимо.

Временная синхронизация обеспечивается через общий календарь. И план, и факт связываются с одной таблицей дат. Это гарантирует, что март в плане и март в факте — это точно один и тот же период. Без общего календаря возможны расхождения из-за разной интерпретации периодов.

Сравнение с альтернативами

В Excel план-факт анализ обычно выполняется через ВПР или ИНДЕКС/ПОИСКПОЗ для сопоставления данных из разных таблиц. Это требует ручного выравнивания периодов и создания промежуточных расчетов. DAX автоматизирует процесс через динамические связи и контекстные вычисления.

SQL-подходы используют JOIN между таблицами плана и факта по ключевым полям. Но SQL-запросы статичны и не адаптируются к изменению фильтров. DAX пересчитывает сопоставление динамически при любом изменении контекста в отчете.

Специализированные EPM-системы (Enterprise Performance Management) предлагают встроенные функции план-факт анализа, но они дороги и требуют отдельной инфраструктуры. DAX предоставляет сопоставимую функциональность в рамках знакомых инструментов Microsoft.

Частые ошибки

Несинхронизированные календари — главная причина ошибок в план- использует финансовый календарь, а факт — календарный, сравнение будет некорректным. Всегда используйте единый календарь или явно выполняйте преобразование периодов.

Игнорирование гранулярности приводит к некорректным сравнениям. Нельзя напрямую сравнивать дневной факт с годовым планом. Необходимо либо агрегировать факт до уровня плана, либо распределить план до уровня факта.

Неправильная обработка BLANK значений искажает отклонения. Если план отсутствует, отклонение должно показывать весь факт как сверхплановый, а не BLANK. Используйте COALESCE или IF(ISBLANK()) для корректной обработки пропусков.

Забывание о знаке отклонений создает путаницу. Положительное отклонение продаж — хорошо, положительное отклонение расходов — плохо. Четко определите и документируйте логику знаков для каждого типа показателей.

```dax

[Actual Sales] :=
SUM(FactInternetSales[SalesAmount])
// План (прошлый год + 20%)
[Budget Sales] :=
CALCULATE(
[Actual Sales] * 1.2,
SAMEPERIODLASTYEAR('DimDate'[FullDateAlternateKey])

[Budget Variance] :=
[Actual Sales] - [Budget Sales]
```

Ответы

Выравнивание обеспечивает корректное сопоставление периодов разной гранулярности между планом и фактом.

TREATAS создает виртуальную связь между несвязанными таблицами бюджета и факта без изменения модели.

Линейное (равномерное) и сезонное (по паттерну прошлого) распределение годового плана по периодам.

Используя COALESCE или условную логику для замены BLANK на 0 где необходимо.

Абсолютное показывает разницу в единицах измерения, процентное — относительное выполнение плана.

Единый календарь гарантирует одинаковую интерпретацию периодов в плане и факте.

Через анализ отклонений от 100% — чем меньше отклонение, тем точнее планирование.
